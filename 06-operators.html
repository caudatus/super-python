파이썬의 연산자

변수와 값을 가지고 계산하고, 비교하고, 조합하는 방법

▸ 이 챕터를 마치면
  • + - * / // % ** 일곱 가지 산술 연산자를 사용할 수 있다
  • == != < > <= >= 비교 연산자로 값을 비교할 수 있다
  • and or not 논리 연산자의 의미를 이해한다
  • += 같은 복합 대입 연산자로 변수를 간결하게 갱신할 수 있다
  • 연산자 우선순위를 이해하고 괄호로 순서를 제어할 수 있다

1.산술 연산자 — 계산하기

산술 연산자는 숫자를 계산할 때 씁니다. 사칙연산은 이미 알고 있지만, 파이썬에는 수학 시간에 못 봤던 연산자가 세 개 더 있습니다. //, %, **입니다. 특히 %는 앞으로 정말 자주 쓰게 될 연산자입니다.

연산자이름예시결과설명
+더하기10 + 313덧셈
-빼기10 - 37뺄셈
*곱하기10 * 330곱셈
/나누기10 / 33.3333…항상 float 반환
//정수 나누기10 // 33몫만 반환 (소수 버림)
%나머지10 % 31나눈 나머지
**거듭제곱2 ** 101024제곱, 세제곱…
python
12345678910
print(10 + 3)    # 13
print(10 - 3)    # 7
print(10 * 3)    # 30
print(10 / 3)    # 3.3333333333333335  ← 항상 소수!
print(10 // 3)   # 3  ← 몫만
print(10 % 3)    # 1  ← 나머지
print(2 ** 10)   # 1024
print(9 ** 0.5)  # 3.0  ← 0.5 거듭제곱 = 제곱근

print(7 / 2)    # 3.5  ← 정수 ÷ 정수도 float

💡 % 연산자 — 왜 자주 쓸까?

나머지(%)는 짝수/홀수 판별에 핵심입니다. n % 2 == 0이면 짝수, n % 2 == 1이면 홀수입니다. 조건문(챕터 18)과 반복문(챕터 21)을 배우면 이 패턴을 수도 없이 쓰게 됩니다.

문자열에도 쓸 수 있는 연산자

+와 *는 문자열에도 쓸 수 있습니다. 숫자와는 다른 방식으로 동작합니다.

python
12345
# 문자열 + 문자열 = 이어붙이기 (concatenation)
print("슈퍼" + "파이썬")    # 슈퍼파이썬

# 문자열 * 정수 = 반복
print("=" * 20)              # ====================
print("안녕! " * 3)          # 안녕!  안녕!  안녕!
실행 결과
슈퍼파이썬
====================
안녕!  안녕!  안녕!

2.비교 연산자 — 두 값 비교하기

비교 연산자는 두 값을 비교해서 항상 True 또는 False를 반환합니다. 조건문(if)에서 핵심적으로 사용됩니다.

연산자의미예시결과
==같다5 == 5True
!=같지 않다5 != 3True
>크다5 > 3True
<작다5 < 3False
>=크거나 같다5 >= 5True
<=작거나 같다3 <= 5True
⚠ == 과 = 는 완전히 다릅니다

=는 값을 저장하는 대입이고, ==는 두 값이 같은지 비교하는 연산자입니다. 이 둘을 헷갈리면 코드가 의도와 다르게 동작합니다. 비교할 때는 반드시 == (등호 두 개)를 써야 합니다.

python
123456789
age = 20

print(age == 20)   # True  — 20과 같은가?
print(age != 18)   # True  — 18이 아닌가?
print(age >= 19)   # True  — 성인인가?
print(age < 18)    # False — 미성년자인가?

# 문자열도 비교할 수 있다
print("python" == "python")  # True
print("Python" == "python")  # False  — 대소문자 구분!
실행 결과

True
False
True

3.논리 연산자 — 조건 조합하기

비교 연산자가 하나의 조건을 만든다면, 논리 연산자는 여러 조건을 조합합니다. "성인이고 학생인가?", "커피 또는 차를 마시는가?" 같은 복합 조건을 표현할 때 씁니다.

연산자의미True가 되는 조건예시결과
and그리고양쪽 모두 TrueTrue and FalseFalse
or또는하나라도 TrueTrue or FalseTrue
not반대원래 값이 Falsenot TrueFalse
python
1234567891011
age       = 22
has_id    = True
is_member = False

# and — 둘 다 True여야 True
print(age >= 19 and has_id)       # True  — 성인이고 신분증 있음

# or — 하나라도 True면 True
print(is_member or age >= 25)      # False — 둘 다 False
print(is_member or age >= 19)      # True  — 하나(age >= 19)가 True

# not — 반대로 뒤집기
print(not is_member)                # True  — "비회원이 아닌가?" → 비회원이므로 True
실행 결과
True
False
True
True
💡 and / or 단락 평가 (Short-circuit)

파이썬은 and에서 앞 조건이 False면 뒤는 확인조차 하지 않습니다. 어차피 전체가 False이기 때문입니다. or도 마찬가지로 앞이 True면 뒤를 건너뜁니다. 지금은 그냥 "그런 게 있구나" 정도만 알아두세요.

4.복합 대입 연산자 — 변수 갱신 단축키

변수의 값을 갱신하는 패턴은 코드에 정말 자주 등장합니다. score = score + 10처럼 쓰는 것을 더 짧게 쓸 수 있는 단축 표현이 있습니다.

단축 표현풀어쓰면예시 (x = 10일 때)결과
+=x = x + nx += 3x → 13
-=x = x - nx -= 3x → 7
*=x = x * nx *= 2x → 20
/=x = x / nx /= 4x → 2.5
//=x = x // nx //= 3x → 3
%=x = x % nx %= 3x → 1
**=x = x ** nx **= 2x → 100
python
123456789
score = 0

score += 10   # score = score + 10 과 같다
print(score)  # 10

score += 5
print(score)  # 15

score *= 2    # 2배로 올리기
print(score)  # 30
실행 결과
10
15
30

+=는 반복문에서 카운터(횟수)를 셀 때, 누적 합계를 구할 때 등 매우 자주 쓰입니다. 익숙해질수록 코드가 훨씬 간결해집니다.

5.연산자 우선순위

여러 연산자가 한 줄에 섞여 있으면 파이썬은 정해진 순서대로 계산합니다. 수학처럼 곱셈이 덧셈보다 먼저 계산되는 것과 같은 원리입니다.

↑ 높은 우선순위 (먼저 계산)
1 ( ) 괄호 — 항상 가장 먼저
2 ** 거듭제곱
3 * / // % 곱셈, 나눗셈 계열
4 + - 덧셈, 뺄셈
5 == != < > <= >= 비교 연산자
6 not 논리 NOT
7 and 논리 AND
8 or 논리 OR — 가장 늦게
↓ 낮은 우선순위 (나중에 계산)
python
12345678
# 우선순위 예시
print(2 + 3 * 4)       # 14  — 곱셈 먼저: 2 + 12
print((2 + 3) * 4)     # 20  — 괄호 먼저: 5 * 4

print(2 ** 3 ** 2)     # 512 — ** 는 오른쪽부터: 2 ** (3**2) = 2**9
print((2 ** 3) ** 2)   # 64  — 괄호로 순서 지정: (8) ** 2

# 헷갈리면 그냥 괄호를 쓰세요!
result = (100 - 20) * 0.9 + 5
print(result)          # 77.0
실행 결과

20
512
64
77.0
💡 우선순위 외울 필요 없습니다

우선순위를 전부 암기하려 하지 마세요. 헷갈리면 항상 괄호를 쓰는 것이 정답입니다. 괄호는 코드를 더 읽기 쉽게 만들기도 합니다. 숙련된 개발자도 복잡한 식에는 괄호를 씁니다.

python — 잘못된 코드

✅ 수정 방법

▸ 핵심 요약
산술 + - * / 기본, // 몫, % 나머지, ** 거듭제곱 / vs // /는 항상 float 반환. //는 소수 버리고 정수 몫만 반환 비교 == != > < >= <= — 결과는 항상 True 또는 False = vs == =는 대입, ==는 비교. 절대 혼동 금지 논리 and 둘 다 True, or 하나라도 True, not 반대 복합 대입 += -= *= 등 — 변수 갱신을 짧게 쓰는 단축 표현 우선순위 괄호 → ** → */% → +- → 비교 → not → and → or. 헷갈리면 괄호로 명시
