기초 디버깅 실습

오류 메시지를 읽고 원인을 파악한 뒤 코드를 고치는 과정을 실전처럼 훈련합니다. 에러를 두려워하지 않는 개발자가 되는 첫걸음입니다.

이 챕터를 마치면
  • 오류 메시지에서 에러 타입과 위치를 빠르게 찾을 수 있다
  • SyntaxError, NameError, TypeError, IndexError를 직접 수정할 수 있다
  • 논리 오류(버그)를 print()로 추적하는 방법을 안다
  • 에러 없이도 잘못 동작하는 코드를 고칠 수 있다
  • 디버깅의 체계적인 5단계 과정을 설명할 수 있다

01디버깅이란?

디버깅(debugging)은 코드에서 오류(버그)를 찾아 고치는 과정입니다. "버그(bug)"라는 단어는 1947년 하버드 대학교에서 실제 나방(moth)이 컴퓨터 회로에 끼어 오작동을 일으킨 사건에서 유래했습니다. 이후 소프트웨어 오류를 버그라 부르게 됐고, 고치는 행위를 디버깅이라 합니다.

처음 코딩을 배울 때 오류 메시지가 뜨면 당황스럽습니다. 하지만 프로 개발자들도 매일 오류를 만납니다. 차이는 단 하나 — 오류 메시지를 읽을 줄 아느냐입니다. 오류 메시지는 파이썬이 여러분에게 보내는 친절한 편지입니다. "이 줄에서 이런 문제가 생겼어요"라고 정확히 알려줍니다.

버그의 두 가지 종류

파이썬의 오류는 크게 두 종류로 나뉩니다.

  • 실행 오류 (Error) — 파이썬이 오류 메시지를 출력하고 프로그램을 멈춥니다. 고맙게도 어디가 잘못됐는지 알려줍니다.
  • 논리 오류 (Logic Bug) — 프로그램이 멈추지 않고 실행은 되지만, 결과가 의도와 다릅니다. 더 찾기 어렵습니다.
💡 오류 메시지는 적이 아닙니다

빨간 오류 메시지를 보면 실패한 것 같은 기분이 듭니다. 하지만 오류 메시지가 없는 논리 오류보다 훨씬 고치기 쉽습니다. 오류 메시지는 파이썬이 여러분의 편에서 정확한 힌트를 주는 것입니다.

02트레이스백(Traceback) 읽는 법 — 복습

26챕터에서 에러 타입을 배웠습니다. 이번엔 오류 메시지 전체(트레이스백)를 위에서 아래로 읽는 순서를 한 번 더 확인합니다.

python
123
scores = [90, 85, 78]
name   = "Alice"
print(name + scores[0])   # ← 오류 발생
TRACEBACK
Traceback (most recent call last):
  File "main.py", line 3, in <module>
    print(name + scores[0])
          ~~~~~~~~~~~~~~~~
TypeError: can only concatenate str (not "int") to str
1
파일명과 줄 번호 확인
File "main.py", line 3 → 3번째 줄에서 문제 발생
2
문제 줄 확인
print(name + scores[0]) → 화살표(~)가 가리키는 부분 집중
3
에러 타입 읽기
TypeError → 타입(종류)이 맞지 않음
4
메시지 해석
can only concatenate str (not "int") to str → 문자열에 정수를 더할 수 없다. str(scores[0])로 변환 필요
💡 트레이스백은 항상 마지막 줄부터

트레이스백이 길게 나오면 당황하지 말고 맨 마지막 두 줄만 먼저 보세요. 에러 타입과 메시지가 가장 중요한 정보입니다. 그다음 line N으로 줄 번호를 확인하세요.

03실전 디버깅 — 케이스 1 : SyntaxError

CASE 1 괄호를 빠뜨린 경우 SyntaxError

아래 코드를 실행하면 SyntaxError가 발생합니다. 오류 메시지를 보고 무엇이 문제인지 생각해보세요.

✗ 오류 코드
python
123
name = "파이썬"
age  = 30
print(f"이름: {name}, 나이: {age}"   # ← 문제가 있습니다
TRACEBACK
  File "main.py", line 3
    print(f"이름: {name}, 나이: {age}"
                                      ^
SyntaxError: '(' was never closed

오류 분석: '(' was never closed — 열린 괄호 (가 닫히지 않았다는 메시지입니다. print( 뒤에 여는 괄호는 있지만 닫는 괄호 )가 없습니다.

✓ 수정된 코드
python
123
name = "파이썬"
age  = 30
print(f"이름: {name}, 나이: {age}")   # 닫는 괄호 ) 추가
핵심 패턴: SyntaxError는 항상 코드를 실행하기 전에 발생합니다. 파이썬이 코드를 읽는 단계에서 문법이 맞지 않으면 바로 멈춥니다. 괄호 () [] {}, 따옴표 "" '', 콜론 :이 빠지는 경우가 가장 흔합니다.

04실전 디버깅 — 케이스 2 : NameError

CASE 2 정의되지 않은 변수를 사용 NameError

점수 합계를 구하는 코드입니다. 실행하면 NameError가 발생합니다.

✗ 오류 코드
python
123456
math_score    = 85
english_score = 90
science_score = 78

total = math_score + english_scroe + science_score  # ← 오타
print(f"총점: {total}")
TRACEBACK
  File "main.py", line 5, in <module>
    total = math_score + english_scroe + science_score
                         ^^^^^^^^^^^^^
NameError: name 'english_scroe' is not defined. Did you mean: 'english_score'?

오류 분석: 'english_scroe' is not defined — english_scroe라는 변수가 없다고 합니다. 자세히 보면 english_score를 english_scroe로 오타 냈습니다. 심지어 파이썬이 Did you mean: 'english_score'?라고 올바른 이름까지 제안해줍니다!

✓ 수정된 코드
python
123456
math_score    = 85
english_score = 90
science_score = 78

total = math_score + english_score + science_score  # scroe → score 수정
print(f"총점: {total}")   # 출력: 총점: 253
핵심 패턴: NameError의 원인은 거의 항상 오타입니다. 에러 메시지가 가리키는 이름을 정확히 확인하고, 선언한 변수 이름과 글자 하나씩 비교하세요. 파이썬 3.10+ 버전부터는 "Did you mean...?" 제안을 많이 제공합니다.

05실전 디버깅 — 케이스 3 : TypeError

CASE 3 문자열과 숫자를 더하려는 경우 TypeError

사용자 나이를 입력받아 10년 후 나이를 출력하는 코드입니다.

✗ 오류 코드
python
123
age = input("나이를 입력하세요: ")   # input()은 항상 문자열 반환
future = age + 10
print(f"10년 후 나이: {future}")
TRACEBACK
  File "main.py", line 2, in <module>
    future = age + 10
             ~~~^~~
TypeError: can only concatenate str (not "int") to str

오류 분석: input()은 언제나 문자열(str)을 반환합니다. 사용자가 25를 입력해도 파이썬은 이를 숫자가 아닌 문자열 "25"로 받습니다. 문자열에 숫자 10을 더하면 TypeError가 발생합니다.

✓ 수정된 코드
python
123
age = int(input("나이를 입력하세요: "))   # int()로 감싸서 정수로 변환
future = age + 10
print(f"10년 후 나이: {future}")   # 출력: 10년 후 나이: 35
핵심 패턴: input() 결과를 숫자 연산에 쓰려면 반드시 int() 또는 float()로 변환하세요. 이것은 파이썬에서 가장 자주 만나는 TypeError 중 하나입니다.
CASE 3-B 반복 횟수에 문자열을 사용 TypeError

같은 TypeError지만 조금 다른 상황입니다. range()에 문자열을 넣으면 어떻게 될까요?

✗ 오류 코드
python
123
n = input("반복 횟수를 입력하세요: ")
for i in range(n):   # n은 문자열 → range()는 정수만 받음
    print(f"반복 {i+1}회")
TRACEBACK
TypeError: 'str' object cannot be interpreted as an integer
✓ 수정된 코드
python
123
n = int(input("반복 횟수를 입력하세요: "))   # int()로 변환
for i in range(n):
    print(f"반복 {i+1}회")

06실전 디버깅 — 케이스 4 : IndexError

CASE 4 리스트 범위를 벗어난 인덱스 IndexError

학생 3명의 성적을 출력하는 코드입니다.

✗ 오류 코드
python
12345
scores = [88, 95, 72]   # 인덱스 0, 1, 2까지만 있음

for i in range(4):        # ← 0, 1, 2, 3 — 범위가 너무 넓음
    print(f"점수 {i+1}: {scores[i]}")
TRACEBACK
  File "main.py", line 4, in <module>
    print(f"점수 {i+1}: {scores[i]}")
                          ~~~~~~~~
IndexError: list index out of range

오류 분석: scores에는 요소가 3개(인덱스 0·1·2)입니다. range(4)는 0·1·2·3 네 번 반복하므로 i = 3일 때 scores[3]에 접근하려 하지만 그 인덱스는 존재하지 않아 IndexError가 발생합니다.

✗ 수정 전
for i in range(4):
    print(scores[i])
✓ 수정 후
for i in range(3):    # 3으로 수정
    print(scores[i])
더 좋은 방법: 리스트 길이에 맞춰 range를 직접 쓰는 것보다, range(len(scores))를 쓰거나 for score in scores:처럼 인덱스 없이 순회하면 IndexError를 아예 방지할 수 있습니다.
python — 더 안전한 방법
1234567
# 방법 1: range(len()) 사용
for i in range(len(scores)):
    print(f"점수 {i+1}: {scores[i]}")

# 방법 2: 직접 순회 (더 파이썬다운 방식)
for score in scores:
    print(f"점수: {score}")

07논리 오류 디버깅 — print()로 추적하기

논리 오류는 프로그램이 실행은 되지만 결과가 의도와 다른 경우입니다. 에러 메시지가 없기 때문에 더 까다롭습니다. 가장 기본적인 방법은 print()를 전략적으로 넣어 변수 값을 추적하는 것입니다.

CASE 5 평균을 잘못 계산하는 경우 논리 오류

1부터 10까지의 합의 평균을 구하는 코드입니다. 실행은 되지만 결과가 이상합니다.

✗ 오류 코드 (에러 없음, 결과만 틀림)
python
123456
total = 0
for i in range(1, 10):    # ← 실수: 10이 포함되지 않음
    total += i

average = total / 10       # ← 실수: 나누는 수도 9여야 함
print(f"평균: {average}")   # 출력: 평균: 4.5 (틀린 답)

문제 추적: 정답은 5.5여야 합니다. print()로 중간 값을 확인해봅니다.

python — print() 디버깅 추가
1234567
total = 0
for i in range(1, 10):
    total += i

print(f"[디버그] total = {total}")   # ← 임시 디버그 출력
# 출력: [디버그] total = 45  ← 예상은 55인데 45가 나옴!
# → range(1, 10)이 1~9만 포함. 10을 빠뜨렸다!
✓ 수정된 코드
python
123456
total = 0
for i in range(1, 11):   # 10 → 11로 수정 (range는 끝값 미포함)
    total += i

average = total / 10
print(f"평균: {average}")   # 출력: 평균: 5.5 ✅
핵심 패턴: 논리 오류가 의심되면 중간중간 print()를 삽입해서 변수 값이 예상대로 변하는지 확인하세요. [디버그] 같은 접두어를 붙이면 나중에 지우기 편합니다. 디버깅이 끝나면 반드시 이 줄들을 제거하세요.
CASE 6 조건문이 의도대로 동작하지 않음 논리 오류

점수가 60점 미만이면 "불합격", 아니면 "합격"을 출력하는 코드입니다. 실행은 되지만 결과가 뒤집혀 있습니다.

✗ 오류 코드
python
12345
score = 45

if score > 60:          # ← >= 이어야 하는데 > 만 씀, 또는 방향이 반대
    print("불합격")
else:
    print("합격")         # 45점인데 "합격"이 출력됨 (의도와 반대)

문제 분석: 조건의 방향이 뒤집혀 있습니다. "60점 미만이면 불합격"이라면 조건은 score < 60이어야 합니다. score > 60이면 45는 60보다 작으므로 else(합격)로 가버립니다.

✓ 수정된 코드
python
12345
score = 45

if score < 60:          # > 를 < 로 수정
    print("불합격")       # 출력: 불합격 ✅
else:
    print("합격")
핵심 패턴: 조건문 논리 오류는 >와 <, >=와 >를 혼동할 때 자주 발생합니다. 조건을 한국어로 먼저 써보고("60 미만이면") 그에 맞는 연산자로 옮기면 실수가 줄어듭니다.

08디버깅 5단계 프로세스

오류를 만났을 때 무작정 코드를 수정하면 오히려 더 꼬이기 쉽습니다. 아래 5단계를 순서대로 따르면 대부분의 오류를 체계적으로 해결할 수 있습니다.

1
침착하게 오류 메시지 읽기
빨간 글씨를 보면 당황하지 말고 맨 마지막 줄부터 읽습니다. 에러 타입과 메시지 → 줄 번호 순서로 확인합니다.
2
해당 줄 집중 분석
line N이 가리키는 줄로 가서 코드를 한 글자씩 살펴봅니다. 괄호·따옴표 쌍이 맞는지, 변수명 오타는 없는지 확인합니다.
3
에러 타입으로 원인 범주 좁히기
SyntaxError → 문법 문제 / NameError → 오타나 미선언 / TypeError → 타입 불일치 / IndexError → 범위 초과. 타입만 알아도 탐색 범위가 크게 줄어듭니다.
4
print()로 중간 값 확인
에러 없이 결과가 이상하다면 의심되는 변수 앞뒤에 print(f"[디버그] 변수명 = {변수명}")을 삽입합니다. 값이 기대와 다른 지점이 버그의 위치입니다.
5
한 번에 하나씩 수정하고 재실행
여러 곳을 한꺼번에 수정하면 어떤 변경이 효과가 있었는지 알 수 없습니다. 한 가지 수정 → 실행 → 확인을 반복하세요. 디버그용 print()는 해결 후 제거합니다.
⚠ 흔한 실수 — 줄 번호를 맹신하지 마세요

오류 메시지가 가리키는 줄에 실제 문제가 없을 때도 있습니다. 특히 SyntaxError는 오류 발생 위치보다 윗줄에 원인이 있는 경우가 많습니다. 예를 들어 3번 줄에 SyntaxError가 나도 실제 원인은 2번 줄의 닫히지 않은 따옴표일 수 있습니다.

💡 에러 메시지를 검색 엔진에 복사하세요

해결이 안 될 때는 에러 메시지 마지막 줄을 그대로 복사해서 구글이나 네이버에 검색하세요. 전 세계 수백만 명의 개발자가 같은 오류를 겪고 해결책을 공유해두었습니다. Stack Overflow가 가장 유용한 자료 사이트입니다.

09직접 고쳐보기 — 3문제

아래 세 코드에는 각각 버그가 숨어 있습니다. 오류 메시지와 코드를 보고 스스로 원인을 찾아보세요. 정답은 드롭다운으로 숨겨두었습니다.

연습 1

python — 이 코드의 버그는 무엇일까요?
1234
items = ["사과", "바나나", "체리"]
count = len(items)
print(f"마지막 항목: {items[count]}")   # 어떤 오류가 날까요?
▶ 정답 보기

오류: IndexError: list index out of range

원인: count = len(items) = 3이지만 리스트 인덱스는 0·1·2까지입니다. items[3]은 존재하지 않습니다.

수정: items[count - 1] 또는 items[-1]로 변경합니다. items[-1]이 파이썬에서 마지막 요소를 가리키는 관용적 표현입니다.

수정된 코드
print(f"마지막 항목: {items[-1]}")  # 출력: 마지막 항목: 체리

연습 2

python — 이 코드의 버그는 무엇일까요?
1234
width  = input("가로 길이: ")
height = input("세로 길이: ")
area   = width * height
print(f"넓이: {area}")
▶ 정답 보기

오류: TypeError: can't multiply sequence by non-int of type 'str'

원인: input()은 항상 문자열을 반환합니다. "5" * "3"처럼 문자열끼리 곱하면 TypeError가 발생합니다. (문자열에 숫자를 곱하면 반복이 되지만, 문자열끼리는 불가)

수정: int()로 변환합니다.

수정된 코드
width  = int(input("가로 길이: "))
height = int(input("세로 길이: "))
area   = width * height
print(f"넓이: {area}")   # 출력: 넓이: 15 (5×3)

연습 3

python — 이 코드의 버그는 무엇일까요? (논리 오류)
1234567
# 홀수만 출력하려는 코드
result = []
for i in range(1, 11):
    if i % 2 == 0:       # ← 이 조건이 홀수인가요, 짝수인가요?
        result.append(i)
print(result)   # 출력: [2, 4, 6, 8, 10]  ← 원하던 결과가 아님
▶ 정답 보기

오류: 에러 없음 — 논리 오류

원인: i % 2 == 0은 짝수 조건입니다. 홀수는 2로 나눈 나머지가 1이므로 i % 2 == 1 또는 i % 2 != 0이어야 합니다.

수정: 조건을 i % 2 != 0으로 변경합니다.

수정된 코드
if i % 2 != 0:   # == 0 → != 0 으로 수정
    result.append(i)
# 출력: [1, 3, 5, 7, 9] ✅
CHAPTER SUMMARY
SyntaxError
문법 오류. 괄호·따옴표·콜론 누락. 실행 전 발생.
NameError
정의되지 않은 변수 사용. 오타가 원인인 경우가 대부분.
TypeError
타입 불일치. input()은 항상 str. 숫자 연산 전 int() 변환.
IndexError
인덱스 범위 초과. len()-1이 마지막 인덱스. -1로 마지막 접근.
논리 오류
에러 없이 결과만 틀림. print()로 중간 값 추적.
