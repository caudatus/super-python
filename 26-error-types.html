에러 읽는 법

SyntaxError · NameError · TypeError · IndexError — 파이썬이 보내는 신호를 정확하게 해석하면, 버그는 더 이상 두렵지 않습니다.

▸ 이 챕터를 마치면
  • 트레이스백(오류 메시지)의 구조를 파악하고 핵심 정보를 빠르게 찾을 수 있다
  • SyntaxError · NameError · TypeError · IndexError 4가지 에러를 구분하고 원인을 설명할 수 있다
  • 에러 메시지만 읽고 어느 줄, 무엇이 문제인지 스스로 진단할 수 있다
  • 각 에러를 유발하는 흔한 실수 패턴을 알고 예방할 수 있다

1.에러는 적이 아니다

코드를 실행했는데 빨간 글씨가 쏟아지면 처음엔 당황스럽습니다. 하지만 에러 메시지는 파이썬이 "여기가 잘못됐어요"라고 알려주는 친절한 신호입니다. 아무 말 없이 조용히 엉뚱한 결과를 내는 버그보다 훨씬 낫습니다.

에러를 무서워하는 대신 읽는 법을 배우면, 디버깅 시간이 반으로 줄어듭니다. 이 챕터에서는 브론즈 과정에서 가장 자주 만나는 4가지 에러를 해부합니다.

💡 TIP

에러가 나면 가장 먼저 메시지의 마지막 줄을 읽으세요. 거기에 에러 유형과 핵심 원인이 담겨 있습니다.

2.트레이스백 해부하기

파이썬이 에러를 발생시키면 트레이스백(Traceback)이라는 오류 보고서가 출력됩니다. 처음 보면 길고 복잡해 보이지만 구조는 항상 일정합니다.

python
12
name = "파이썬"
print(neme)   # 변수 이름을 잘못 입력 (name → neme)
TRACEBACK
Traceback (most recent call last):
  File "main.py", line 2, in <module>
    print(neme)
          ^^^^
NameError: name 'neme' is not defined
TRACEBACK 읽는 순서
어디서 File "main.py", line 2 — 파일 이름과 몇 번째 줄인지 알려줍니다. 가장 먼저 확인할 정보입니다.
어떤 코드 print(neme) — 문제가 발생한 코드 줄을 보여줍니다. ^^^^ 기호가 정확한 위치를 가리킵니다.
에러 유형 NameError — 에러의 종류입니다. 이 이름만 보고 원인의 범주를 파악할 수 있습니다.
에러 메시지 name 'neme' is not defined — 에러의 구체적인 원인입니다. 가장 중요한 정보입니다.
해석 "2번 줄의 neme이라는 이름이 정의되지 않았다" → name을 neme으로 잘못 입력했구나!
ℹ 참고

트레이스백의 "Traceback (most recent call last)"은 "가장 최근에 호출된 코드부터 역순으로 보여준다"는 뜻입니다. 함수 안에서 에러가 나면 여러 줄이 쌓이지만, 항상 마지막 줄의 에러 유형과 메시지가 핵심입니다.

3.SyntaxError — 문법이 틀렸어요

SyntaxError 문법 오류 코드 실행 전 발생

문법(Syntax)이 틀렸을 때 발생합니다. 파이썬은 코드를 실행하기 전에 먼저 문법을 검사하는데, 규칙을 어긴 부분을 발견하면 바로 알려줍니다. 마치 국어 시험에서 맞춤법이 틀린 문장을 빨간 펜으로 표시하는 것과 같습니다.

흔한 원인: 콜론(:) 누락, 괄호 미닫힘, 따옴표 미닫힘, 들여쓰기 오류

예시 1 — 콜론 누락

python
12
if x > 0          # ← 콜론(:)이 없음
    print("양수")
오류 메시지
  File "main.py", line 1
    if x > 0
            ^
SyntaxError: expected ':'

해결: if x > 0:처럼 콜론을 붙입니다. if, elif, else, for, while 뒤에는 항상 콜론이 옵니다.

예시 2 — 괄호 미닫힘

python
1
print("안녕하세요"   # ← 닫는 ) 없음
오류 메시지
  File "main.py", line 1
    print("안녕하세요"
         ^
SyntaxError: '(' was never closed

해결: 여는 괄호와 닫는 괄호의 수가 같은지 확인합니다. print("안녕하세요")처럼 닫는 괄호를 추가합니다.

예시 3 — 따옴표 미닫힘

python
1
name = "파이썬   # ← 닫는 " 없음
오류 메시지
SyntaxError: EOL while scanning string literal

해결: 문자열을 시작한 따옴표와 같은 종류로 닫아야 합니다. "파이썬"처럼 닫는 따옴표를 추가합니다.

⚠ 주의

SyntaxError가 발생한 줄 번호가 실제 실수 위치와 다를 때가 있습니다. 오류의 원인은 표시된 줄보다 앞에 있는 경우가 많으므로, 지시된 줄과 그 바로 앞 줄을 함께 확인하세요.

4.NameError — 이 이름을 모르겠어요

NameError 이름 오류 해당 코드 실행 시 발생

존재하지 않는 변수나 함수 이름을 사용했을 때 발생합니다. 파이썬은 변수 이름을 정확히 기억하므로, 철자 하나만 달라도 "이런 이름은 모른다"고 합니다.

흔한 원인: 변수명 오타, 대소문자 혼동(Name vs name), 선언 전에 사용

예시 1 — 변수명 오타

python
12
score = 85
print(scroe)   # score → scroe 오타
오류 메시지
NameError: name 'scroe' is not defined

해결: 메시지에 나온 이름 'scroe'을 확인하고 score로 수정합니다.

예시 2 — 선언 전 사용

python
12
print(total)   # total을 아직 만들지 않았음
total = 100   # 선언은 이 줄에서 됨
오류 메시지
NameError: name 'total' is not defined

해결: 변수를 사용하기 전에 반드시 먼저 값을 할당해야 합니다. 코드는 위에서 아래로 순서대로 실행됩니다.

예시 3 — 대소문자 혼동

python
12
Name = "김철수"       # 대문자 N으로 선언
print(name)          # 소문자 n으로 사용 → 다른 변수!
오류 메시지
NameError: name 'name' is not defined

해결: 파이썬은 대소문자를 구분합니다. Name, name, NAME은 모두 다른 변수입니다. 선언할 때와 사용할 때의 이름을 반드시 일치시키세요.

💡 TIP — NameError 예방법

변수 이름은 소문자 + 언더스코어 방식(snake_case)으로 일관되게 짓는 것이 좋습니다. playerScore 보다 player_score처럼 쓰면 대소문자 오타가 자연스럽게 줄어듭니다.

5.TypeError — 타입이 맞지 않아요

TypeError 타입 오류 해당 코드 실행 시 발생

잘못된 타입(자료형)으로 연산하거나 함수를 호출했을 때 발생합니다. 숫자와 문자열을 +로 더하려 하는 경우가 대표적입니다.

흔한 원인: 문자열 + 숫자 연산, input() 결과를 변환 없이 계산, 함수 인수 개수 불일치

예시 1 — 문자열과 숫자를 더함

python
12
age = "20"       # 따옴표가 있으니 문자열 "20"
print(age + 5)   # 문자열 + 정수 → TypeError
오류 메시지
TypeError: can only concatenate str (not "int") to str

해결: 숫자 계산이 목적이라면 int(age) + 5, 문자열 연결이 목적이라면 age + str(5)를 사용합니다. 타입을 통일하면 됩니다.

예시 2 — input() 결과를 바로 계산

input()은 사용자가 무엇을 입력하든 항상 문자열을 반환합니다. 숫자를 입력해도 문자열 "5"로 받습니다.

python
1234567
# ❌ 잘못된 예
n = input("숫자를 입력하세요: ")
print(n + 10)   # "5" + 10 → TypeError!

# ✅ 올바른 예
n = int(input("숫자를 입력하세요: "))   # int()로 감쌈
print(n + 10)   # 5 + 10 = 15 ✓
올바른 코드 실행 결과 (5 입력 시)
15
⚠ 황금 규칙

input()으로 받은 값을 숫자로 다루려면 반드시 int() 또는 float()으로 감싸야 합니다. 이를 잊어서 생기는 TypeError는 브론즈 과정에서 가장 흔한 실수입니다.

6.IndexError — 범위를 벗어났어요

IndexError 인덱스 오류 해당 코드 실행 시 발생

리스트나 문자열에서 존재하지 않는 인덱스에 접근할 때 발생합니다. 리스트의 길이를 초과하는 위치를 요청하거나, 빈 리스트에서 값을 꺼내려 할 때 주로 나타납니다.

흔한 원인: 인덱스를 1부터 세는 착각, 리스트 길이와 마지막 인덱스 혼동

예시 1 — 인덱스 범위 초과

python
12
fruits = ["사과", "바나나", "딸기"]   # 길이 3, 유효 인덱스 0~2
print(fruits[3])                         # 인덱스 3은 없음
오류 메시지
IndexError: list index out of range

해결: 리스트의 인덱스는 0부터 시작합니다. 길이가 3이면 유효한 인덱스는 0, 1, 2입니다. len(fruits) - 1이 마지막 인덱스입니다.

python
12345
fruits = ["사과", "바나나", "딸기"]
# 인덱스:    0       1          2
print(fruits[0])    # "사과" (첫 번째)
print(fruits[2])    # "딸기" (마지막)
print(fruits[-1])   # "딸기" (음수 인덱스로도 접근 가능)
실행 결과
사과
딸기
딸기

예시 2 — 빈 리스트 접근

python
12
items = []         # 빈 리스트
print(items[0])   # 원소가 하나도 없음
오류 메시지
IndexError: list index out of range

해결: 리스트에 접근하기 전에 if len(items) > 0:으로 비어 있지 않은지 확인하는 습관을 들이세요.

💡 TIP — 마지막 항목에 안전하게 접근하기

리스트나 문자열의 마지막 원소는 [-1]로 접근할 수 있습니다. 길이를 몰라도 안전합니다. fruits[-1]은 항상 마지막 과일을 가리킵니다. [-2]는 뒤에서 두 번째입니다.

7.4가지 에러 한눈에 비교

에러 유형 발생 시점 핵심 원인 해결 방향
SyntaxError 실행 전 문법 규칙 위반 콜론·괄호·따옴표 확인
NameError 실행 중 존재하지 않는 이름 사용 오타·대소문자·선언 순서 확인
TypeError 실행 중 타입 불일치 연산 int() / str() / float() 변환
IndexError 실행 중 인덱스 범위 초과 인덱스는 0부터, 마지막은 len()-1
ℹ 이 밖의 에러들

파이썬에는 ValueError(잘못된 값), ZeroDivisionError(0으로 나누기), AttributeError(없는 메서드 호출) 등 다양한 에러가 있습니다. 이들은 각 개념을 배울 때 함께 다루겠습니다. 지금은 이 4가지만 확실히 익혀두세요.

⌨ 직접 해보기 — 에러를 유발하고 읽어보기

아래 코드들을 IDLE이나 VS Code에서 직접 실행하고, 에러 메시지를 보고 유형과 원인을 스스로 진단하는 연습을 해보세요.

01. print("안녕"을 실행하고 SyntaxError를 확인하세요. 괄호를 닫아서 해결해보세요.
02. print(Hello)를 실행하고 NameError를 확인하세요. 따옴표를 추가해 문자열로 고쳐보세요.
03. n = input("숫자: ")로 입력받고 print(n + 10)을 실행해보세요. TypeError가 뜨면 int()로 감싸서 해결하세요.
04. colors = ["빨강", "파랑"]을 만들고 colors[5]를 실행하세요. IndexError를 확인하고 유효한 인덱스 범위를 적어보세요.
도전. 실행하기 전에 예측해보세요: nums = [1, 2, 3]; print(nums[len(nums)]) — 어떤 에러가 날까요? 왜일까요?
▸ 핵심 요약
트레이스백 읽기 마지막 줄 = 에러 유형 + 원인. "File ..., line N" = 문제 위치. SyntaxError 문법 오류. 콜론·괄호·따옴표 누락. 실행 전에 검출됨. NameError 존재하지 않는 이름. 오타·대소문자 혼동·선언 전 사용. TypeError 타입 불일치. input()은 항상 문자열 → 계산 전 int() 필수. IndexError 인덱스 범위 초과. 인덱스는 0부터, 마지막은 len()-1. 디버깅 4단계 ① 에러 유형 확인 → ② 줄 번호 이동 → ③ 메시지 해석 → ④ 수정
