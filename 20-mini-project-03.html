1.프로젝트 소개

MINI PROJECT ③
맞춤형 여행 추천 프로그램
예산, 선호 날씨, 동행 유형을 입력받아 여러 조건을 조합한 뒤, 딱 맞는 국내 여행지와 여행 팁을 추천해주는 프로그램입니다. 조건문의 진짜 힘 — 여러 if / elif / else와 and / or를 조합해 복잡한 판단 로직을 만들어봅니다.
if / elif / else and / or / not 비교 연산자 input() int() f-string

2.실행 결과 미리보기

완성된 프로그램은 아래처럼 동작합니다. 사용자가 입력한 조건에 따라 매번 다른 결과가 출력됩니다. 밑줄 친 부분이 사용자 입력입니다.

실행 예시 1 — 따뜻한 날씨, 커플, 중간 예산

터미널 
====================================
   🗺️  맞춤형 여행 추천 프로그램
====================================
여행 예산을 입력하세요 (만원 단위): 25
선호하는 날씨를 입력하세요 (따뜻함/시원함/상관없음): 따뜻함
동행 유형을 입력하세요 (혼자/커플/가족/친구): 커플

====================================
   ✈️  추천 여행지: 제주도
====================================
 📍 예산       : 25만원 (적합)
 🌤️ 날씨       : 따뜻한 날씨 선호
 👫 동행       : 커플
------------------------------------
 💡 여행 팁
    · 협재 해변과 한라산 코스를 묶어서 2박 3일로 계획하세요.
    · 렌터카를 이용하면 이동이 훨씬 편합니다.
    · 성산일출봉은 일출 시간에 맞춰 방문하면 특별한 추억이 됩니다.
====================================

터미널 
여행 예산을 입력하세요 (만원 단위): 12
선호하는 날씨를 입력하세요 (따뜻함/시원함/상관없음): 시원함
동행 유형을 입력하세요 (혼자/커플/가족/친구): 가족

====================================
   ✈️  추천 여행지: 강원도 평창
====================================
 📍 예산       : 12만원 (타이트)

------------------------------------
 💡 여행 팁
    · 대관령 양떼 목장은 아이들이 특히 좋아합니다.
    · 숙박은 펜션보다 캠핑을 활용하면 예산을 크게 줄일 수 있습니다.
    · 오대산 월정사는 무료 입장이라 가족 나들이에 딱입니다.
====================================

실행 예시 3 — 잘못된 날씨 입력

터미널 
여행 예산을 입력하세요 (만원 단위): 30
선호하는 날씨를 입력하세요 (따뜻함/시원함/상관없음): 맑음
동행 유형을 입력하세요 (혼자/커플/가족/친구): 혼자

💡 추천 여행지 결정 기준

날씨 선호와 동행 유형의 조합으로 여행지가 결정됩니다. 예산은 여행지를 바꾸지 않고, 예산 코멘트와 팁의 내용을 달리합니다. 아래 표를 참고하되, 창의적으로 구성해도 좋습니다.

날씨동행추천 여행지

따뜻함가족 / 친구부산
시원함혼자 / 친구강릉
시원함커플 / 가족강원도 평창
상관없음혼자경주
상관없음커플 / 가족 / 친구전주

코드를 바로 짜기 전에, 어떤 기능이 필요한지 파악하세요. "무엇을 만드는가"를 먼저 이해하는 것이 코딩보다 중요합니다.

구분기능사용 개념

4.단계별 구현 가이드

처음부터 완성본을 만들려 하지 말고, 한 단계씩 완성하고 직접 실행해서 결과를 확인한 뒤 다음으로 넘어가세요.

STEP 1 헤더 출력과 세 가지 입력 받기 print · input · int

프로그램 시작 배너를 출력하고, 예산 / 날씨 / 동행 유형을 차례로 입력받습니다. 예산은 int()로 변환해서 숫자로 저장하세요.

STEP 1 구현
print("====================================")
print("   🗺️  맞춤형 여행 추천 프로그램")
print("====================================")
budget  = int(input("여행 예산을 입력하세요 (만원 단위): "))
weather = input("선호하는 날씨를 입력하세요 (따뜻함/시원함/상관없음): ")
group   = input("동행 유형을 입력하세요 (혼자/커플/가족/친구): ")

▶ 실행하면 배너와 입력 프롬프트 세 개가 차례로 나타나야 합니다.

STEP 2 입력값 유효성 검사하기 if · or · not

날씨와 동행 유형이 허용된 값인지 확인합니다. 잘못된 값이 들어오면 오류 메시지를 출력하고 프로그램을 멈춥니다.

핵심 아이디어: weather가 "따뜻함", "시원함", "상관없음" 중 하나가 아니라면 잘못된 입력입니다. and와 !=를 조합하거나, not in을 사용해보세요.

STEP 2 구현 — and + != 방식
if weather != "따뜻함" and weather != "시원함" and weather != "상관없음":
    print("⚠️  날씨 입력이 올바르지 않습니다. '따뜻함', '시원함', '상관없음' 중에서 입력해주세요.")
# 동행 유형도 같은 방식으로 검사해보세요
💡 더 파이썬다운 방법 — not in

if weather not in ["따뜻함", "시원함", "상관없음"]:처럼 리스트로 묶어서 검사할 수도 있습니다. 지금 단계에서는 어떤 방법을 써도 괜찮습니다.

▶ 잘못된 값을 입력했을 때 오류 메시지가 출력되는지 확인하세요.

STEP 3 날씨 + 동행 조합으로 여행지 결정하기 if / elif / else + and / or

이 단계가 이 프로젝트의 핵심입니다. 날씨와 동행 유형을 동시에 검사해서 여행지를 결정합니다.

and로 두 조건을 함께 묶고, or로 여러 선택지를 처리합니다. 조건의 순서가 중요합니다 — 더 구체적인 조건을 앞에 쓰세요.

STEP 3 구현 — 조건 조합 구조
if weather == "따뜻함" and (group == "혼자" or group == "커플"):
    destination = "제주도"
    tips = [
        "협재 해변과 한라산 코스를 묶어서 2박 3일로 계획하세요.",
        "렌터카를 이용하면 이동이 훨씬 편합니다.",
        "성산일출봉은 일출 시간에 맞춰 방문하면 특별한 추억이 됩니다."
    ]
elif weather == "따뜻함" and (group == "가족" or group == "친구"):
    destination = "부산"
    tips = [
        "해운대와 광안리를 모두 방문해보세요.",
        "자갈치 시장 해산물은 꼭 드세요!",
        "감천 문화마을은 인생 사진 명소입니다."
    ]
# 나머지 조건들도 같은 패턴으로 채워보세요...

각 분기마다 destination 변수와 tips 리스트를 설정합니다. 이렇게 해두면 출력 코드를 한 번만 작성해도 됩니다.

▶ 여섯 가지 조합이 모두 올바른 여행지를 출력하는지 하나씩 확인해보세요.

STEP 4 예산 코멘트 결정하기 if / else · 비교 연산자

예산에 따라 적합 여부를 코멘트로 달아줍니다. budget의 크기를 비교해서 문자열을 결정합니다.

STEP 4 구현
if budget >= 30:
    budget_comment = "넉넉한 여행 가능"
elif budget >= 20:
    budget_comment = "적합"
elif budget >= 10:
    budget_comment = "타이트"
else:
    budget_comment = "매우 빠듯 — 당일치기 추천"

이렇게 destination, tips, budget_comment 세 가지 변수를 미리 결정해두면, 출력 코드가 훨씬 깔끔해집니다.

STEP 5 결과 출력하기 f-string · print()

STEP 3, 4에서 결정한 변수들을 사용해서 정리된 결과를 출력합니다. tips는 리스트이므로 인덱싱으로 꺼내세요.

STEP 5 구현 — 출력 형식
print()
print("====================================")
print(f"   ✈️  추천 여행지: {destination}")
print("====================================")
print(f" 📍 예산       : {budget}만원 ({budget_comment})")

# 날씨 아이콘도 조건으로 결정할 수 있습니다
if weather == "따뜻함":
    weather_icon = "🌤️"
elif weather == "시원함":
    weather_icon = "❄️"
else:
    weather_icon = "🌈"

print(f" {weather_icon} 날씨       : {weather} 날씨 선호")
print(f" 👤 동행       : {group}")
print("------------------------------------")
print(" 💡 여행 팁")
print(f"    · {tips[0]}")
print(f"    · {tips[1]}")
print(f"    · {tips[2]}")
print("====================================")

▶ 여러 조합으로 실행해서 여행지와 팁이 올바르게 바뀌는지 확인하세요.

5.막혔을 때 — 단계별 힌트

충분히 혼자 시도해본 뒤에 힌트를 열어보세요.

💡 힌트 1 — and와 or, 언제 괄호가 필요한가요? ▼

and는 or보다 우선순위가 높습니다. 그래서 A and B or C는 (A and B) or C로 해석됩니다.

원하는 의미가 A and (B or C)라면 반드시 괄호를 써야 합니다.

예시
# 이렇게 쓰면 의도와 다를 수 있습니다
if weather == "따뜻함" and group == "혼자" or group == "커플":  # ❌

# 괄호로 명확하게 의도를 표현하세요
if weather == "따뜻함" and (group == "혼자" or group == "커플"):  # ✅
💡 힌트 2 — 유효성 검사를 어디에 넣어야 하나요? ▼

유효성 검사는 여행지 결정 로직보다 앞에 두세요. 잘못된 입력이면 뒤의 코드가 실행될 필요가 없습니다.

지금 단계에서는 오류 메시지를 출력하고 프로그램이 그냥 끝나는 방식으로 구현하면 됩니다. 나중에 while문을 배우면 재입력을 받을 수 있습니다.

구조 예시
# 1. 입력 받기
budget  = ...
weather = ...
group   = ...

# 2. 유효성 검사 (잘못된 입력이면 여기서 멈춤)
if weather != "따뜻함" and ...:
    print("⚠️ ...")
else:
    # 3. 정상 처리 — 여행지 결정 + 출력
    if weather == "따뜻함" and ...:
        ...
💡 힌트 3 — elif 조건이 6개나 되는데, 더 간결하게 쓸 방법은 없나요? ▼

날씨를 먼저 if/elif로 나누고, 그 안에서 동행을 다시 if/elif로 나누는 중첩 조건문을 쓰면 구조가 더 명확해집니다.

중첩 조건문 구조
if weather == "따뜻함":
    if group == "혼자" or group == "커플":
        destination = "제주도"
        tips = [...]
    else:     # 가족 또는 친구
        destination = "부산"
        tips = [...]
elif weather == "시원함":
    ...

두 방식 모두 맞습니다. 직접 해보고 더 읽기 편한 쪽을 선택하세요.

💡 힌트 4 — 동행 아이콘도 조건으로 바꾸고 싶어요 ▼

동행 유형에 맞는 이모지를 if/elif/else로 결정할 수 있습니다. 출력 코드보다 앞에서 미리 결정해두세요.

동행 아이콘 예시
if group == "혼자":
    group_icon = "🧳"
elif group == "커플":
    group_icon = "👫"
elif group == "가족":
    group_icon = "👨‍👩‍👧‍👦"
else:
    group_icon = "👯"

print(f" {group_icon} 동행       : {group}")

6.전체 정답 코드

스스로 완성하지 못했다면 정답 코드를 열어보세요. 단순히 복사하지 말고, 한 줄씩 읽으며 왜 이렇게 짰는지 이해하는 것이 중요합니다.

20-travel-recommender.py
# ══════════════════════════════════════
# 맞춤형 여행 추천 프로그램
# ══════════════════════════════════════

# 1. 헤더 출력
print("====================================")
print("   🗺️  맞춤형 여행 추천 프로그램")
print("====================================")

# 2. 입력 받기
budget  = int(input("여행 예산을 입력하세요 (만원 단위): "))
weather = input("선호하는 날씨를 입력하세요 (따뜻함/시원함/상관없음): ")
group   = input("동행 유형을 입력하세요 (혼자/커플/가족/친구): ")

# 3. 유효성 검사
#    날씨와 동행 입력이 잘못되면 오류 메시지를 출력하고 else 블록 전체를 건너뜀
valid_weather = ["따뜻함", "시원함", "상관없음"]
valid_group   = ["혼자", "커플", "가족", "친구"]

if weather not in valid_weather:
    print("⚠️  날씨 입력이 올바르지 않습니다. '따뜻함', '시원함', '상관없음' 중에서 입력해주세요.")
elif group not in valid_group:
    print("⚠️  동행 유형이 올바르지 않습니다. '혼자', '커플', '가족', '친구' 중에서 입력해주세요.")
else:
    # 4. 날씨 + 동행 조합으로 여행지와 팁 결정
    if weather == "따뜻함":
        if group == "혼자" or group == "커플":
            destination = "제주도"
            tips = [
                "협재 해변과 한라산 코스를 묶어서 2박 3일로 계획하세요.",
                "렌터카를 이용하면 이동이 훨씬 편합니다.",
                "성산일출봉은 일출 시간에 맞춰 방문하면 특별한 추억이 됩니다."
            ]
        else:   # 가족 또는 친구
            destination = "부산"
            tips = [
                "해운대와 광안리를 모두 방문해보세요.",
                "자갈치 시장 해산물은 꼭 드세요!",
                "감천 문화마을은 인생 사진 명소입니다."
            ]
    elif weather == "시원함":
        if group == "혼자" or group == "친구":
            destination = "강릉"
            tips = [
                "경포대 해변에서 새벽 일출을 꼭 보세요.",
                "안목 해변 커피거리는 현지인 추천 명소입니다.",
                "버스와 자전거를 조합하면 시내 이동이 편합니다."
            ]
        else:   # 커플 또는 가족
            destination = "강원도 평창"
            tips = [
                "대관령 양떼 목장은 아이들이 특히 좋아합니다.",
                "숙박은 펜션보다 캠핑을 활용하면 예산을 크게 줄일 수 있습니다.",
                "오대산 월정사는 무료 입장이라 가족 나들이에 딱입니다."
            ]
    else:   # 상관없음
        if group == "혼자":
            destination = "경주"
            tips = [
                "첨성대, 불국사, 석굴암을 하루 동선으로 묶어보세요.",
                "자전거 대여로 시내 고분군을 천천히 돌아보는 것을 추천합니다.",
                "황남빵은 경주 여행의 필수 기념품입니다."
            ]
        else:   # 커플, 가족, 친구
            destination = "전주"
            tips = [
                "한옥마을에서 한복 체험을 해보세요. 다양한 대여점이 있습니다.",
                "전주 비빔밥은 골목 식당이 유명 맛집보다 맛있을 때가 많습니다.",
                "오목대에서 내려다보는 한옥마을 전경이 일품입니다."
            ]

    # 5. 예산 코멘트 결정
    if budget >= 30:
        budget_comment = "넉넉한 여행 가능"
    elif budget >= 20:
        budget_comment = "적합"
    elif budget >= 10:
        budget_comment = "타이트"
    else:
        budget_comment = "매우 빠듯 — 당일치기 추천"

    # 6. 날씨/동행 아이콘 결정
    if weather == "따뜻함":
        weather_icon = "🌤️"
    elif weather == "시원함":
        weather_icon = "❄️"
    else:
        weather_icon = "🌈"

    if group == "혼자":
        group_icon = "🧳"
    elif group == "커플":
        group_icon = "👫"
    elif group == "가족":
        group_icon = "👨‍👩‍👧‍👦"
    else:
        group_icon = "👯"

    # 7. 결과 출력
    print()
    print("====================================")
    print(f"   ✈️  추천 여행지: {destination}")
    print("====================================")
    print(f" 📍 예산       : {budget}만원 ({budget_comment})")
    print(f" {weather_icon} 날씨       : {weather} 날씨 선호")
    print(f" {group_icon} 동행       : {group}")
    print("------------------------------------")
    print(" 💡 여행 팁")
    print(f"    · {tips[0]}")
    print(f"    · {tips[1]}")
    print(f"    · {tips[2]}")
    print("====================================")

📌 핵심 패턴 — "먼저 결정, 나중에 출력": 여행지(destination), 팁(tips), 예산 코멘트(budget_comment), 아이콘들을 각각 변수에 저장해두고, 출력 코드는 맨 마지막에 한 번만 작성합니다. 이 패턴을 쓰면 조건 로직과 출력 로직이 분리되어 코드가 훨씬 읽기 쉬워집니다.
