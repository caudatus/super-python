예외처리 기초

프로그램은 언제든 예상치 못한 상황을 만납니다. 존재하지 않는 파일, 잘못된 입력, 0으로 나누기… try / except는 이런 상황을 프로그램이 죽지 않고 우아하게 처리하는 방법입니다.

► 이 챕터를 마치면
  • try / except의 기본 구조를 이해하고 작성할 수 있다
  • 예외 종류에 따라 다른 except 절로 분기 처리할 수 있다
  • else와 finally의 역할을 구분하고 활용할 수 있다
  • 예외 메시지를 as e로 받아 출력할 수 있다
  • 파일 입출력과 예외처리를 결합한 안전한 코드를 작성할 수 있다

1.왜 예외처리가 필요한가?

코드를 완벽하게 짜도 실행 중 예측 불가능한 상황은 반드시 생깁니다. 사용자가 숫자 대신 문자를 입력하거나, 읽으려는 파일이 없거나, 리스트 범위를 벗어나는 경우처럼요. 이때 예외처리 없이 그냥 두면 프로그램이 즉시 종료됩니다.

PYTHON — 예외처리 없는 코드의 위험
# 사용자가 "abc"를 입력하면?
num = int(input("숫자를 입력하세요: "))
print(f"입력한 숫자: {num}")
► 입력: abc
ValueError: invalid literal for int() with base 10: 'abc' 프로그램 종료 ☠

예외처리를 쓰면 오류가 발생해도 프로그램을 계속 실행시키거나, 친절한 메시지를 보여줄 수 있습니다.

2.try / except 기본 구조

파이썬의 예외처리는 네 개의 키워드로 구성됩니다: try, except, else, finally.

try:←오류가 날 수 있는 코드를 감싸는 블록
int(input("입력: "))←여기서 오류 발생 가능
except ValueError:←ValueError 발생 시 실행
print("숫자를 입력하세요")
else:←오류 없이 성공했을 때 실행 (선택)
print("성공!")
finally:←오류 여부와 관계없이 항상 실행 (선택)
print("끝")
PYTHON — 가장 기본적인 예외처리
 1try:
 2    num = int(input("숫자를 입력하세요: "))
 3    print(f"입력한 숫자의 2배: {num * 2}")
 4except ValueError:
 5    print("❌ 올바른 숫자를 입력해주세요")
► 입력: abc
❌ 올바른 숫자를 입력해주세요 프로그램이 계속 실행됩니다 ✔

3.실행 흐름 이해하기

try 블록 안에서 오류가 발생하면 즉시 해당 줄에서 실행을 멈추고 맞는 except 절로 점프합니다.

try 블록 실행
↓
오류 발생
except 절 실행
|
오류 없음
else 절 실행
↓
finally 절 실행 (항상)
↓
다음 코드 계속 실행
PYTHON — 흐름 확인 예제
 1def safe_divide(a, b):
 2    try:
 3        result = a / b         # b가 0이면 ZeroDivisionError
 4        print(f"try 블록 완료: {result}")
 5    except ZeroDivisionError:
 6        print("except 실행: 0으로 나눌 수 없습니다")
 7        result = 0
 8    else:
 9        print("else 실행: 나눗셈 성공!")
10    finally:
11        print("finally 실행: 항상 여기까지 옵니다")
12    return result
13
14print("--- 정상 케이스 ---")
15safe_divide(10, 2)
16print("--- 오류 케이스 ---")
17safe_divide(10, 0)
► 실행 결과
--- 정상 케이스 --- try 블록 완료: 5.0 else 실행: 나눗셈 성공! finally 실행: 항상 여기까지 옵니다 --- 오류 케이스 --- except 실행: 0으로 나눌 수 없습니다 finally 실행: 항상 여기까지 옵니다
💡 else와 finally 언제 쓸까?

else — 오류가 없을 때만 실행할 후속 작업에 사용. 성공한 경우만 로그 기록 등
finally — 파일·네트워크 연결처럼 오류 여부와 관계없이 반드시 닫아야 할 자원 정리에 사용

4.여러 예외 종류 처리하기

하나의 try 블록에서 여러 종류의 오류가 날 수 있습니다. except를 여러 개 쓰거나, 괄호로 묶어 한꺼번에 처리할 수 있습니다.

PYTHON — 다중 except
 1def get_item(lst, index_str):
 2    try:
 3        idx = int(index_str)    # ValueError 가능
 4        return lst[idx]          # IndexError 가능
 5    except ValueError:
 6        print(f"  ❌ '{index_str}'는 숫자가 아닙니다")
 7    except IndexError:
 8        print(f"  ❌ 인덱스 {index_str}이 범위를 벗어났습니다")
 9    return None
10
11data = ["사과", "바나나", "포도"]
12print(get_item(data, "1"))    # 바나나
13get_item(data, "abc")         # ValueError
14get_item(data, "10")          # IndexError
15
16# 여러 예외를 한꺼번에 처리 — 튜플로 묶기
17try:
18    int("abc")
19except (ValueError, TypeError):
20    print("값 오류 또는 타입 오류")
► 실행 결과
바나나 ❌ 'abc'는 숫자가 아닙니다 ❌ 인덱스 10이 범위를 벗어났습니다 값 오류 또는 타입 오류

5.예외 메시지 받기 — as e

예외가 발생했을 때 파이썬이 만들어주는 오류 메시지를 변수로 받아 출력하거나 로그로 남길 수 있습니다.

PYTHON
 1try:
 2    result = 10 / 0
 3except ZeroDivisionError as e:
 4    print(f"오류 발생: {e}")     # division by zero
 5    print(f"오류 타입: {type(e).__name__}")  # ZeroDivisionError
 6
 7# 여러 예외 + as e
 8try:
 9    score = int("백점")
10except ValueError as e:
11    print(f"입력 오류: {e}")
12    # invalid literal for int() with base 10: '백점'
13
14# 모든 예외를 잡는 Exception (마지막 안전망)
15try:
16    {}["없는키"]
17except Exception as e:
18    print(f"예상치 못한 오류 [{type(e).__name__}]: {e}")
► 실행 결과
오류 발생: division by zero 오류 타입: ZeroDivisionError 입력 오류: invalid literal for int() with base 10: '백점' 예상치 못한 오류 [KeyError]: '없는키'
⚠️ except Exception은 마지막 수단

모든 예외를 잡는 except Exception은 진짜 버그도 덮어버릴 수 있습니다. 구체적인 예외 타입을 먼저 처리하고, Exception은 예상치 못한 예외를 로그에 남기는 마지막 안전망으로만 쓰세요.

6.자주 마주치는 예외 종류

예외언제 발생하나예시 코드
ValueError값 형식이 맞지 않을 때int("abc")
TypeError타입이 맞지 않을 때"3" + 3
ZeroDivisionError0으로 나눌 때10 / 0
IndexError리스트 범위 초과[1,2,3][10]
KeyError딕셔너리에 없는 키{"a":1}["b"]
FileNotFoundError파일이 없을 때open("없음.txt")
AttributeError없는 속성/메서드 호출(42).upper()
NameError정의되지 않은 변수 사용print(x) (x 없음)

7.파일 입출력과 예외처리 결합

파일을 다룰 때는 예외처리가 필수입니다. 파일이 없거나 권한이 없는 경우가 흔히 발생하기 때문입니다. finally로 파일을 반드시 닫는 패턴도 중요합니다.

PYTHON — 안전한 파일 읽기
 1# 방법 1: try/except/finally 직접 사용
 2def read_file_safe(filename):
 3    f = None
 4    try:
 5        f = open(filename, "r", encoding="utf-8")
 6        content = f.read()
 7        print(f"파일 읽기 성공: {len(content)}자")
 8        return content
 9    except FileNotFoundError:
10        print(f"❌ '{filename}' 파일을 찾을 수 없습니다")
11        return None
12    except PermissionError:
13        print(f"❌ '{filename}' 읽기 권한이 없습니다")
14        return None
15    finally:
16        if f:
17            f.close()   # 오류가 나도 반드시 파일 닫기
18            print("파일 닫음")
19
20read_file_safe("없는파일.txt")

with문 — finally 없이도 자동으로 파일을 닫아주는 패턴

with open(...) as f: 구문은 블록이 끝나면 예외 발생 여부와 관계없이 자동으로 파일을 닫아줍니다. 파일 작업에서 가장 권장되는 패턴입니다.

PYTHON — with문 + 예외처리 (권장 패턴)
 1def read_file(filename):
 2    try:
 3        with open(filename, "r", encoding="utf-8") as f:
 4            return f.read()   # with 블록 끝에 자동으로 파일 닫힘
 5    except FileNotFoundError:
 6        print(f"파일 없음: {filename}")
 7        return None
 8
 9def write_file(filename, content):
10    try:
11        with open(filename, "w", encoding="utf-8") as f:
12            f.write(content)
13        print(f"✅ '{filename}' 저장 완료")
14    except IOError as e:
15        print(f"저장 실패: {e}")
💡 with문이 finally보다 좋은 이유

파일을 열 때 with를 쓰면 finally: f.close()를 매번 쓸 필요가 없고, 실수로 닫는 것을 잊는 실수도 방지됩니다. 파이썬에서 파일 작업의 표준 패턴입니다.

8.실전 예제 — 안전한 점수 입력기

사용자 입력을 받는 현실적인 프로그램에서 예외처리가 어떻게 작동하는지 확인해보세요.

PYTHON — 점수 입력 검증 함수
 1def get_valid_score(subject):
 2    """유효한 점수(0~100)를 입력받을 때까지 반복합니다."""
 3    while True:
 4        try:
 5            raw = input(f"{subject} 점수 입력 (0-100): ")
 6            score = int(raw)           # ValueError 가능
 7            if not (0 <= score <= 100):
 8                raise ValueError("점수는 0~100 사이여야 합니다")
 9        except ValueError as e:
10            print(f"  ❌ 입력 오류: {e}")
11        else:
12            return score              # 오류 없이 성공하면 반환
13
14# 사용 예 (실제 실행 시 입력받음)
15# math = get_valid_score("수학")
16# print(f"수학 점수: {math}")
17
18# 시뮬레이션 — 입력 시나리오
19test_inputs = ["abc", "150", "87"]
20for inp in test_inputs:
21    try:
22        score = int(inp)
23        if not (0 <= score <= 100):
24            raise ValueError("점수는 0~100 사이여야 합니다")
25        print(f"  ✅ 유효한 점수: {score}")
26    except ValueError as e:
27        print(f"  ❌ [{inp}] → {e}")
► 실행 결과
❌ [abc] → invalid literal for int() with base 10: 'abc' ❌ [150] → 점수는 0~100 사이여야 합니다 ✅ 유효한 점수: 87
📌 raise — 예외를 직접 발생시키기

8번 줄의 raise ValueError("...")처럼 조건이 맞지 않을 때 예외를 직접 발생시킬 수 있습니다. 범위 검사, 유효성 검사에서 자주 쓰이는 패턴입니다.

9.자주 하는 실수와 해결법

❌
try 블록이 너무 큼 — 수십 줄을 하나의 try로 감싸면 어디서 오류가 났는지 알기 어렵습니다.
오류 가능성이 있는 최소한의 코드만 try로 감싸세요.
❌
except: 만 쓰기 — except: (예외 타입 없이)는 KeyboardInterrupt까지 잡아 Ctrl+C로도 프로그램을 못 끄게 만듭니다. 최소한 except Exception:을 쓰세요.
❌
except에서 아무것도 안 하기 — except: pass는 오류를 조용히 삼켜 버려 디버깅을 매우 어렵게 만듭니다. 최소한 print나 로그를 남기세요.
❌
except 순서 실수 — 상위 예외(예: Exception)를 앞에 쓰면 하위 예외가 절대 실행되지 않습니다. 구체적인 예외를 먼저, 넓은 예외를 나중에 쓰세요.
✓
올바른 패턴 — 구체적 예외 → 일반 예외 순서, try는 최소 범위, except에는 반드시 처리 로직 작성
📌 핵심 요약
try — 오류 날 수 있는 코드
except 예외 — 오류 발생 시 처리
else — 오류 없이 성공 시 실행
finally — 항상 실행 (자원 정리)
except E as e — 오류 메시지 받기
여러 except — 구체적 순서로 작성
raise — 예외 직접 발생
with open() — 자동 파일 닫기
try는 최소 범위로 감싸기
except에는 반드시 처리 로직 작성
