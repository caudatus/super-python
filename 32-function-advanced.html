함수 심화

기본값 매개변수, *args, **kwargs — 함수를 훨씬 유연하고 강력하게 만드는 세 가지 도구를 배웁니다.

▸ 이 챕터를 마치면
  • 기본값 매개변수로 선택적 인자를 가진 함수를 만들 수 있다
  • *args로 개수가 정해지지 않은 인자를 받는 함수를 작성할 수 있다
  • **kwargs로 키워드 인자를 딕셔너리로 받을 수 있다
  • 함수에서 여러 값을 한꺼번에 반환하는 방법을 이해한다

1.기본값 매개변수 — 생략해도 되는 인자

함수를 만들 때 매개변수에 기본값을 미리 설정해두면, 호출할 때 그 인자를 생략할 수 있습니다. 마치 카페 메뉴에서 "아이스 라떼, 사이즈 기본(M)"처럼 따로 말하지 않으면 기본값이 적용되는 것과 같습니다.

python
12345678
def greet(name, greeting="안녕하세요"):
    print(f"{greeting}, {name}님!")

# greeting을 생략하면 기본값 "안녕하세요" 사용
greet("철수")
# greeting을 직접 전달하면 기본값을 덮어씀
greet("영희", "좋은 아침이에요")
greet("민준", "반가워요")
실행 결과
안녕하세요, 철수님!
좋은 아침이에요, 영희님!
반가워요, 민준님!

기본값 매개변수는 파이썬 내장 함수에서도 자주 볼 수 있습니다. 예를 들어 print()의 end="\n"이 바로 기본값 매개변수입니다. 우리가 따로 지정하지 않으면 줄바꿈이 기본으로 붙는 이유가 여기 있습니다.

실용 예시 — 기본 단위를 설정한 계산기

python
12345678
def shipping_cost(weight, rate=100):
    # 기본 요금: 1kg당 100원
    return weight * rate

print(shipping_cost(3))        # 300 (rate=100 기본값)
print(shipping_cost(3, 150))   # 450 (rate=150 직접 지정)
print(shipping_cost(5))        # 500 (rate=100 기본값)
실행 결과
300
450
500
⚠ 규칙: 기본값 매개변수는 반드시 뒤쪽에

기본값이 있는 매개변수는 기본값이 없는 매개변수 뒤쪽에 와야 합니다. 그렇지 않으면 SyntaxError가 납니다.

python
1234
# ✗ 잘못된 순서 (SyntaxError)
def wrong(greeting="안녕", name): ...

# ✓ 올바른 순서
def right(name, greeting="안녕"): ...

2.*args — 인자를 몇 개든 받기

인자를 몇 개 받을지 미리 알 수 없을 때는 *args를 사용합니다. 별표(*)가 "여러 개를 묶어라"는 신호입니다. 함수 안에서 args는 튜플로 전달됩니다.

💡 이름은 관례일 뿐

*args에서 args는 관례적인 이름이고, 실제 중요한 것은 앞의 *입니다. *numbers, *items처럼 더 명확한 이름을 써도 완전히 동일하게 동작합니다.

python
12345678910
def total(*numbers):
    # numbers는 전달된 인자 전체를 담은 튜플
    result = 0
    for n in numbers:
        result += n
    return result

print(total(1, 2))              # 3
print(total(10, 20, 30))       # 60
print(total(1, 2, 3, 4, 5))  # 15
실행 결과
3
60
15

일반 매개변수와 함께 쓰기

*args는 일반 매개변수와 함께 쓸 수 있습니다. 이때 *args는 고정 매개변수 뒤쪽에 위치합니다.

python
12345678
def introduce(name, *hobbies):
    print(f"이름: {name}")
    print(f"취미: {', '.join(hobbies)}")

introduce("철수", "독서")
introduce("영희", "그림", "피아노", "요리")
실행 결과
이름: 철수
취미: 독서
이름: 영희
취미: 그림, 피아노, 요리

3.**kwargs — 이름이 있는 인자를 몇 개든 받기

*args가 위치 기반 인자를 여러 개 받는다면, **kwargs는 키워드 인자를 여러 개 받습니다. 함수 안에서 kwargs는 딕셔너리가 됩니다. 키워드 인자란 함수이름(키=값) 형태로 이름을 지정해서 전달하는 인자입니다.

💡 이름은 관례일 뿐

**kwargs도 **가 핵심이고, kwargs는 관례명입니다. **options, **data처럼 바꿔도 됩니다.

python
12345678910
def print_profile(**info):
    # info는 전달된 키워드 인자를 담은 딕셔너리
    for key, value in info.items():
        print(f"{key}: {value}")

print_profile(name="철수", age=20)
print("---")
print_profile(name="영희", age=22, city="서울", job="개발자")
실행 결과
name: 철수
age: 20
---
name: 영희
age: 22
city: 서울
job: 개발자

세 가지를 한꺼번에 — 매개변수 순서

일반 매개변수, *args, **kwargs를 모두 쓸 때는 반드시 아래 순서를 지켜야 합니다.

python
123456789
# 순서: 일반 → *args → **kwargs
def order_food(table, *items, **options):
    print(f"테이블 {table}번")
    print(f"주문: {', '.join(items)}")
    for k, v in options.items():
        print(f"  {k}: {v}")

order_food(3, "파스타", "콜라", spicy="약하게", takeout=False)
실행 결과
테이블 3번
주문: 파스타, 콜라
  spicy: 약하게
  takeout: False
종류기호함수 안에서의 타입전달 방식
일반 매개변수name단일 값f("철수")
가변 위치 인자*args튜플f(1, 2, 3)
가변 키워드 인자**kwargs딕셔너리f(a=1, b=2)

4.여러 값 반환하기

파이썬 함수는 return으로 값을 하나만 돌려보낼 수 있지만, 쉼표로 구분해서 여러 개를 나열하면 자동으로 튜플로 묶어 반환합니다. 이를 이용하면 여러 결과를 한꺼번에 돌려받을 수 있습니다.

python
12345678910111213
def min_max(numbers):
    lowest  = numbers[0]
    highest = numbers[0]
    for n in numbers:
        if n < lowest:  lowest  = n
        if n > highest: highest = n
    return lowest, highest   # 튜플로 자동 묶임

result = min_max([3, 7, 1, 9, 5])
print(result)          # (1, 9) — 튜플로 받음

# 언패킹으로 각각 받기 (더 자주 쓰는 방식)
low, high = min_max([3, 7, 1, 9, 5])
print(f"최솟값: {low}, 최댓값: {high}")
실행 결과
(1, 9)
최솟값: 1, 최댓값: 9

실용 예시 — 통계 함수

python
1234567891011
def stats(scores):
    total   = 0
    for s in scores: total += s
    average = total / len(scores)
    return total, average, len(scores)

data = [85, 92, 78, 96, 88]
t, avg, cnt = stats(data)
print(f"합계: {t}, 평균: {avg}, 인원: {cnt}")
실행 결과
합계: 439, 평균: 87.8, 인원: 5
ℹ 언패킹(unpacking)이란?

low, high = min_max(...)처럼 튜플의 각 값을 여러 변수에 한꺼번에 대입하는 것을 언패킹이라 합니다. 반환값의 개수와 변수 개수가 같아야 합니다. 이 패턴은 파이썬 코드에서 매우 자주 등장합니다.
