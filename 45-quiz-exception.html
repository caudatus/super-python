Q1.

아래 코드의 실행 결과를 예상하세요. try에서 예외가 발생할 때 except 블록이 실행됩니다.

try: result = 10 / 0 except ZeroDivisionError: print("0으로 나눌 수 없습니다")
출력:0으로 나눌 수 없습니다

Q2.

숫자를 입력받아 정수로 변환하는 코드입니다. 사용자가 "abc"를 입력했을 때 발생하는 예외 이름을 쓰세요.

num = int(input("숫자 입력: ")) # 입력: "abc"
예외:ValueError
HINT

int("abc")처럼 숫자로 변환할 수 없는 값을 넘길 때 발생하는 오류입니다. "값 오류"를 영어로?

ANSWER
# ValueError # int()에 숫자로 변환 불가능한 문자열을 넘길 때 발생 # 예: int("abc"), int("3.14"), int("")

# 수정 전 — 오류 발생 가능 num = int("hello") print(num)
출력:잘못된 입력입니다

Q4.

아래 코드의 실행 결과를 순서대로 쓰세요. finally 블록은 예외 발생 여부와 관계없이 항상 실행됩니다.

출력:A C D
HINT

A 출력 후 ZeroDivisionError 발생 → B는 건너뜀 → C 실행 → finally는 무조건 D 실행.

A # try 블록 첫 줄 C # 예외 발생 → except 블록 D # finally는 항상 실행 # "B"는 예외 발생 후라 실행되지 않음

else는 try 블록에서 예외가 발생하지 않았을 때만 실행됩니다. 10/2는 정상 계산되므로…

ANSWER
# 10 / 2 = 5.0 → 예외 없음 → else 블록 실행 결과: 5.0 # else는 try 블록이 예외 없이 완료됐을 때만 실행

nums = [1, 2, 3] print(nums[10]) # 범위 초과

try: result = int("파이썬") except ValueError as e: print(f"오류: {e}")

HINT

as e로 예외 객체를 변수에 담으면 str(e)로 오류 메시지를 볼 수 있습니다. f-string 안에 {e}를 쓰면 자동으로 문자열로 변환됩니다.

ANSWER
# ValueError 발생, e에 예외 객체 저장 오류: invalid literal for int() with base 10: '파이썬'
Q8.

아래 중 finally 블록에 넣기에 가장 적합한 코드를 고르고 이유를 쓰세요.

A. print("결과 계산 중...") B. result = num * 2 C. print("프로그램 종료") D. if num > 0: print("양수")
답:C (항상 실행돼야 할 정리 작업)
HINT

finally는 예외 발생 여부와 무관하게 항상 실행됩니다. 파일 닫기, 로그 기록, 종료 메시지처럼 반드시 실행돼야 하는 정리 작업에 씁니다.

ANSWER
# C — "프로그램 종료" # finally는 성공/실패 모두에서 실행되는 정리 코드에 적합 # A, D는 try 안에, B는 예외 없을 때만 실행 → else 블록에 적합

# KeyError # d.get("age")를 사용하면 None을 반환하고 예외가 발생하지 않음 # d.get("age", 0)처럼 기본값을 지정할 수도 있음

Q20.

Q22.

# 요구사항 # - KeyError 발생 시 기본값 반환 # - 정상 실행 시 값 반환 # - 항상 "조회 완료" 출력 data = {"name": "Alice", "age": 30} # safe_get(data, "name") → "Alice", "조회 완료" # safe_get(data, "city") → "없음", "조회 완료"
출력:Alice 조회 완료 없음 조회 완료
HINT

try에서 d[key], except KeyError에서 기본값, else에서 값 출력, finally에서 "조회 완료".

def safe_get(d, key, default="없음"): try: value = d[key] except KeyError: value = default finally: print("조회 완료") return value data = {"name": "Alice", "age": 30} print(safe_get(data, "name")) print(safe_get(data, "city"))
Q25.

숫자 리스트를 입력받아 각 원소를 int로 변환하되, 변환 실패한 항목은 건너뛰고 성공한 것만 새 리스트로 반환하는 함수를 작성하세요.

inputs = ["10", "abc", "30", "xyz", "50"]
출력:[10, 30, 50]
HINT

루프 안에서 try/except ValueError를 사용하세요. 성공하면 리스트에 추가, 실패하면 continue 또는 그냥 건너뜁니다.

ANSWER
def safe_convert(items): result = [] for item in items: try: result.append(int(item)) except ValueError: pass # 변환 실패 → 건너뜀 return result inputs = ["10", "abc", "30", "xyz", "50"] print(safe_convert(inputs)) # [10, 30, 50]
Q26.

파일을 열어 숫자를 읽고 합계를 계산하는 코드를 작성하세요. 파일이 없으면 "파일 없음"을, 숫자가 아닌 줄은 건너뜁니다.

# nums.txt 예시 10 abc 30 xyz 60
출력:합계: 100
HINT

외부 try/except FileNotFoundError로 파일 열기, 내부 루프에서 각 줄마다 try/except ValueError로 변환 시도.

ANSWER
total = 0 try: with open("nums.txt", "r", encoding="utf-8") as f: for line in f: try: total += int(line.strip()) except ValueError: pass # 숫자 아닌 줄 건너뜀 except FileNotFoundError: print("파일 없음") else: print(f"합계: {total}")

Q28.

아래 코드를 읽고 try / except / else / finally 네 블록이 각각 어떤 역할을 하는지 설명하세요.

try: num = int(input("숫자: ")) result = 100 / num except ValueError: print("숫자를 입력하세요") except ZeroDivisionError: print("0 입력 불가") else: print(f"100 / {num} = {result}") finally: print("계산 종료")
핵심 키워드:try:실행 / except:오류처리 / else:성공시 / finally:항상
HINT

try → 예외 가능성 있는 코드 / except → 예외 종류별 처리 / else → 예외 없을 때만 / finally → 무조건 마지막

ANSWER
# try : 예외가 발생할 수 있는 코드를 감쌈 # except: 발생한 예외 종류에 따라 처리 # else : try 블록이 예외 없이 완료됐을 때 실행 # finally: 예외 발생 여부와 관계없이 항상 실행 (정리)
Q29.

나이를 입력받아 검증하는 함수를 작성하세요. 아래 모든 조건을 처리해야 합니다.

  • 숫자가 아닌 입력 → ValueError 처리 → "숫자를 입력하세요"
  • 음수 → raise ValueError → "나이는 0 이상이어야 합니다"
  • 150 초과 → raise ValueError → "현실적인 나이를 입력하세요"
  • 정상 → "나이: {age}세"
  • 항상 → "입력 처리 완료"
테스트:validate_age("abc") → "숫자를 입력하세요" + "입력 처리 완료" validate_age("-1") → "나이는 0 이상이어야 합니다" + "입력 처리 완료" validate_age("25") → "나이: 25세" + "입력 처리 완료"
HINT

try 안에서 int() 후 범위 검증 → raise ValueError. except에서 메시지 출력. else에서 정상 출력. finally에서 항상 출력.

ANSWER
def validate_age(user_input): try: age = int(user_input) if age < 0: raise ValueError("나이는 0 이상이어야 합니다") if age > 150: raise ValueError("현실적인 나이를 입력하세요") except ValueError as e: print(e if "나이" in str(e) else "숫자를 입력하세요") else: print(f"나이: {age}세") finally: print("입력 처리 완료")
Q30.

🏆 종합 도전 문제 — 안전한 성적 계산기를 완성하세요.

  • scores.txt 파일을 읽어 숫자를 파싱합니다
  • 파일이 없으면 "파일 없음" 출력 후 종료
  • 각 줄이 숫자가 아니면 경고 출력 후 건너뜀
  • 점수가 0~100 범위 밖이면 ValueError를 raise
  • 유효한 점수의 평균을 출력
  • 항상 "처리 완료"를 출력
scores.txt:85 abc 92 -10 78
출력:경고: 'abc' 숫자 아님, 건너뜀 경고: '-10' 범위 초과, 건너뜀 평균: 85.0 처리 완료
HINT

① 외부 try/except FileNotFoundError → ② 루프 안 try/except ValueError → ③ int 변환 → ④ 범위 체크 → raise → ⑤ finally에서 "처리 완료"

ANSWER
scores = [] try: with open("scores.txt", "r", encoding="utf-8") as f: for line in f: raw = line.strip() if not raw: continue try: score = int(raw) if score < 0 or score > 100: raise ValueError(f"범위 초과") scores.append(score) except ValueError as e: if "범위" in str(e): print(f"경고: '{raw}' 범위 초과, 건너뜀") else: print(f"경고: '{raw}' 숫자 아님, 건너뜀") except FileNotFoundError: print("파일 없음") else: if scores: print(f"평균: {sum(scores)/len(scores)}") finally: print("처리 완료")
