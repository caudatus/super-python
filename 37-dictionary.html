딕셔너리 — 이름표가 달린 서랍장

리스트가 순서로 데이터를 찾는다면, 딕셔너리는 이름(키)으로 데이터를 찾습니다. 연락처, 학생 정보, 설정값처럼 "이름 → 값" 형태의 데이터를 다룰 때 가장 강력한 도구입니다.

▸ 이 챕터를 마치면
  • 딕셔너리를 만들고 키로 값에 접근할 수 있다
  • 항목을 추가·수정·삭제하는 방법을 안다
  • keys(), values(), items()로 딕셔너리를 순회할 수 있다
  • get()으로 안전하게 값을 조회할 수 있다
  • 리스트와 딕셔너리의 차이를 정확히 설명할 수 있다

1.딕셔너리란?

리스트는 데이터를 순서(인덱스)로 관리합니다. 예를 들어 학생 정보를 리스트로 만들면 [0]이 이름인지 나이인지 코드를 읽지 않으면 알 수 없습니다.

PYTHON — 리스트 방식의 불편함
# 리스트: 순서로만 구분 → 무슨 값인지 기억해야 함
student = ["김파이썬", 17, "서울고", 95.5]
print(student[3])  # 95.5 — 이게 성적인지 어떻게 알지?

딕셔너리는 데이터에 이름표(키, key)를 붙입니다. 마치 서랍장에 "이름", "나이", "학교" 라벨을 붙여두는 것처럼요.

PYTHON — 딕셔너리 방식
# 딕셔너리: 이름표(키)로 구분 → 의미가 명확함
student = {
    "이름": "김파이썬",
    "나이": 17,
    "학교": "서울고",
    "성적": 95.5
}
print(student["성적"])  # 95.5 — 이름만 봐도 바로 이해!
▸ 딕셔너리 구조 시각화
"이름":"김파이썬"str
"나이":17int
"학교":"서울고"str
"성적":95.5float
📌 딕셔너리 핵심 용어

키(key) — 이름표. 문자열·숫자·튜플처럼 변경 불가능한 타입만 사용 가능
값(value) — 실제 데이터. 어떤 타입이든 OK (리스트, 딕셔너리 중첩도 가능)
키:값 쌍(key-value pair) — 딕셔너리의 기본 단위

2.딕셔너리 만들기

딕셔너리는 중괄호 {}와 콜론 :으로 만듭니다.

PYTHON
 1# 기본 생성
 2person = {"name": "Alice", "age": 25}
 3
 4# 여러 줄로 작성 (항목이 많을 때 읽기 쉬움)
 5cafe = {
 6    "name":     "파이썬 카페",
 7    "city":     "서울",
 8    "rating":   4.8,
 9    "open":     True
10}
11
12# 빈 딕셔너리
13empty = {}
14
15# dict() 함수로 생성
16scores = dict(math=90, english=85, science=92)
17print(scores)  # {'math': 90, 'english': 85, 'science': 92}
💡 파이썬 3.7+ 딕셔너리는 순서가 보장됩니다

예전 파이썬(3.6 이하)에서는 딕셔너리의 순서가 뒤섞였습니다. 지금은 입력한 순서대로 유지되므로 안심하고 사용하세요.

3.값 읽기 — 대괄호 vs get()

딕셔너리에서 값을 꺼내는 방법은 두 가지입니다. 차이를 꼭 알아두세요.

PYTHON
 1menu = {"아메리카노": 4500, "라떼": 5000, "녹차": 4000}
 2
 3# ① 대괄호 [] — 키가 없으면 KeyError 발생!
 4print(menu["라떼"])       # 5000
 5# print(menu["홍차"])   # KeyError: '홍차' ← 위험!
 6
 7# ② get() — 키가 없으면 None 반환 (안전)
 8print(menu.get("녹차"))    # 4000
 9print(menu.get("홍차"))    # None ← 오류 없이 None 반환
10
11# get()에 기본값 지정 — 키가 없을 때 대신 반환할 값
12price = menu.get("홍차", "메뉴 없음")
13print(price)               # 메뉴 없음
▸ 실행 결과
5000 4000 None 메뉴 없음
방법키가 있을 때키가 없을 때추천 상황
dict[key]값 반환KeyError 발생키가 반드시 있다고 확신할 때
dict.get(key)값 반환None 반환키 존재 여부가 불확실할 때
dict.get(key, 기본값)값 반환기본값 반환없을 때 대체값이 필요할 때

4.추가 · 수정 · 삭제

딕셔너리는 생성 후에도 자유롭게 데이터를 바꿀 수 있습니다.

PYTHON
 1profile = {"name": "Bob", "age": 20}
 2
 3# 추가 — 없는 키에 값을 대입하면 새로 생성
 4profile["city"] = "부산"
 5print(profile)  # {'name': 'Bob', 'age': 20, 'city': '부산'}
 6
 7# 수정 — 있는 키에 값을 대입하면 덮어씀
 8profile["age"] = 21
 9print(profile["age"])   # 21
10
11# 삭제 — del 키워드
12del profile["city"]
13print(profile)  # {'name': 'Bob', 'age': 21}
14
15# pop() — 삭제하면서 값을 반환
16name = profile.pop("name")
17print(name)      # Bob
18print(profile)  # {'age': 21}
⚠️ 추가와 수정은 문법이 똑같습니다

dict[key] = value 는 키가 이미 있으면 수정, 없으면 추가입니다. 실수로 기존 데이터를 덮어쓰지 않도록 주의하세요.

5.딕셔너리 순회 — keys() · values() · items()

딕셔너리를 for문으로 순회할 때 세 가지 방법이 있습니다. 상황에 맞게 골라 쓰세요.

PYTHON
 1scores = {"수학": 90, "영어": 85, "과학": 92}
 2
 3# ① keys() — 키만 순회 (기본 for문과 동일)
 4for subject in scores.keys():
 5    print(subject)
 6# 수학  영어  과학
 7
 8# ② values() — 값만 순회
 9for score in scores.values():
10    print(score)
11# 90  85  92
12
13# ③ items() — (키, 값) 쌍을 동시에 — 가장 많이 씁니다!
14for subject, score in scores.items():
15    print(f"{subject}: {score}점")
16# 수학: 90점
17# 영어: 85점
18# 과학: 92점

키 존재 여부 확인 — in 연산자

in으로 키가 딕셔너리에 있는지 미리 확인할 수 있습니다.

PYTHON
 1menu = {"아메리카노": 4500, "라떼": 5000}
 2
 3if "라떼" in menu:
 4    print(f"라떼 가격: {menu['라떼']}원")
 5
 6if "홍차" not in menu:
 7    print("홍차는 메뉴에 없습니다")
▸ 실행 결과
라떼 가격: 5000원 홍차는 메뉴에 없습니다

6.자주 쓰는 딕셔너리 메서드

메서드설명예시
d.get(k, 기본값)키 k의 값 반환, 없으면 기본값d.get("나이", 0)
d.keys()모든 키 반환for k in d.keys()
d.values()모든 값 반환for v in d.values()
d.items()모든 (키, 값) 쌍 반환for k, v in d.items()
d.pop(k)키 k 삭제 후 값 반환val = d.pop("이름")
d.update(d2)딕셔너리 d2의 내용을 d에 병합d.update({"나이": 20})
len(d)키:값 쌍의 개수len(d)

update() — 딕셔너리 병합

PYTHON
 1base = {"name": "Alice", "age": 25}
 2extra = {"city": "서울", "age": 26}  # age 키가 겹침
 3
 4base.update(extra)
 5print(base)
 6# {'name': 'Alice', 'age': 26, 'city': '서울'}
 7# age는 extra의 값 26으로 덮어써짐

7.리스트 vs 딕셔너리 — 언제 뭘 쓸까?

📋 리스트
✅순서가 중요한 데이터
✅같은 종류의 항목 여러 개
✅인덱스(0, 1, 2...)로 접근
📌예: 학생 이름 목록, 점수 리스트
🗂️ 딕셔너리
✅속성이 여러 개인 하나의 객체
✅이름(키)으로 빠르게 조회
✅데이터의 의미가 명확해야 할 때
📌예: 한 학생의 정보, 설정값, JSON
💡 선택 기준 한 줄 요약

"같은 것을 여러 개" 담으면 리스트, "하나에 대한 여러 정보"를 담으면 딕셔너리

8.실전 예제 — 학생 성적 관리

지금까지 배운 것을 조합해서 성적을 관리하는 간단한 프로그램을 만들어봅니다.

PYTHON
 1# 학생 성적 딕셔너리
 2scores = {
 3    "김파이썬": 92,
 4    "이코딩":   87,
 5    "박디버그":  76,
 6    "최알고리":  95
 7}
 8
 9# 전체 출력
10print("=== 성적 현황 ===")
11for name, score in scores.items():
12    grade = "A" if score >= 90 else "B" if score >= 80 else "C"
13    print(f"  {name}: {score}점  ({grade}등급)")
14
15# 평균 계산
16avg = sum(scores.values()) / len(scores)
17print(f"\n평균: {avg:.1f}점")
18
19# 특정 학생 조회
20target = "이코딩"
21result = scores.get(target, "없는 학생")
22print(f"\n{target}의 성적: {result}")
23
24# 새 학생 추가
25scores["정루프"] = 88
26print(f"\n전체 학생 수: {len(scores)}명")
▸ 실행 결과
=== 성적 현황 === 김파이썬: 92점 (A등급) 이코딩: 87점 (B등급) 박디버그: 76점 (C등급) 최알고리: 95점 (A등급) 평균: 87.5점 이코딩의 성적: 87 전체 학생 수: 5명
💡 sum() + values() 조합

sum(dict.values())는 딕셔너리의 모든 값을 더하는 파이썬다운 표현입니다. 평균, 합계를 구할 때 자주 씁니다.

9.중첩 딕셔너리 — 딕셔너리 안의 딕셔너리

딕셔너리의 값으로 다른 딕셔너리를 넣을 수 있습니다. 여러 명의 정보를 한 번에 관리할 때 유용합니다.

PYTHON
 1students = {
 2    "김파이썬": {"나이": 17, "성적": 92, "도시": "서울"},
 3    "이코딩":   {"나이": 18, "성적": 87, "도시": "부산"},
 4}
 5
 6# 접근: 외부 키 → 내부 키 순서로
 7print(students["김파이썬"]["성적"])   # 92
 8print(students["이코딩"]["도시"])     # 부산
 9
10# 전체 순회
11for name, info in students.items():
12    print(f"{name} — {info['나이']}세, {info['도시']}, {info['성적']}점")
▸ 실행 결과
92 부산 김파이썬 — 17세, 서울, 92점 이코딩 — 18세, 부산, 87점

10.자주 발생하는 오류와 해결법

❌
KeyError — 존재하지 않는 키를 []로 조회할 때
d["없는키"] → d.get("없는키") 또는 "없는키" in d 로 먼저 확인
❌
키 타입 오류 — 리스트를 키로 사용하려 할 때
d[[1,2]] = "값" → TypeError (리스트는 변경 가능하므로 키 불가)
튜플 d[(1,2)] = "값" 은 OK
❌
중복 키 — 같은 키를 두 번 쓰면 나중 것만 남음
{"a": 1, "a": 2} → {"a": 2} (오류는 안 나지만 의도치 않은 결과)
✅
안전한 조회 패턴 — get()을 기본으로 쓰고, 반드시 있어야 하는 키만 [] 사용
📌 핵심 요약
{키: 값} 형태로 생성
d[key] — 없으면 KeyError
d.get(key, 기본값) — 안전 조회
d[key] = val — 추가/수정
del d[key] / d.pop(key) — 삭제
.keys() / .values() / .items()
key in d — 존재 확인
d.update(d2) — 병합
값으로 딕셔너리 중첩 가능
키는 변경 불가 타입만 허용
