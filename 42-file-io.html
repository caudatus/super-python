파일 입출력 — 데이터를 영구적으로 저장하기

지금까지 만든 모든 데이터는 프로그램이 끝나면 사라졌습니다. 파일을 사용하면 데이터를 디스크에 저장해 다음에도 다시 불러올 수 있습니다.

▸ 학습 목표
  • open()으로 파일을 열고 닫는 원리를 이해한다
  • with 문으로 파일을 안전하게 다루는 방법을 익힌다
  • 텍스트 파일을 읽고(r), 쓰고(w), 추가하는(a) 세 가지 모드를 구별한다
  • 여러 줄 파일을 줄 단위로 처리하는 패턴을 손에 익힌다

1.왜 파일이 필요한가?

파이썬 프로그램이 실행되는 동안 모든 데이터는 메모리(RAM)에 올라갑니다. 프로그램이 종료되는 순간 메모리의 내용은 전부 지워집니다. 사용자가 입력한 이름도, 계산한 점수도, 저장한 메모도 — 모두 사라집니다.

반면 파일(File)은 디스크(SSD/HDD)에 저장됩니다. 프로그램이 꺼져도 내용이 그대로 남아있죠. 일기장에 글을 써두면 다음날 꺼내 읽을 수 있는 것처럼, 파일에 데이터를 쓰면 나중에 다시 읽을 수 있습니다.

ℹ 메모리 vs 파일

메모리: 빠르지만 임시 저장. 전원이 꺼지면 사라짐.
파일: 느리지만 영구 저장. 프로그램이 꺼져도 남아있음.

이번 챕터에서는 텍스트 파일(.txt)을 다루는 방법을 배웁니다. CSV, JSON 같은 포맷도 원리는 동일하므로 텍스트 파일을 완전히 이해하면 나머지는 응용에 불과합니다.

2.파일 열기 — open()

파일을 다루는 첫 단계는 파일을 여는 것입니다. 현실에서 책을 읽으려면 먼저 책을 펴야 하는 것처럼, 파이썬도 파일을 읽거나 쓰기 전에 반드시 open()으로 파일을 열어야 합니다.

python
1
파일객체 = open("파일이름.txt", "모드", encoding="utf-8")

open()은 세 가지 정보를 받습니다.

매개변수역할예시
"파일이름.txt"열 파일의 이름(경로)"diary.txt", "data/scores.txt"
"모드"파일을 어떻게 열지 결정"r", "w", "a" 등
encoding텍스트 인코딩 방식한글 파일이면 반드시 "utf-8"

파일 모드 3가지

모드는 파일을 어떤 목적으로 여는지를 나타냅니다. 가장 중요한 세 가지를 익혀두세요.

모드이름설명
"r"읽기 (read)파일을 읽습니다. 파일이 없으면 오류 발생. 기본값.
"w"쓰기 (write)파일에 새로 씁니다. 파일이 있으면 내용을 완전히 덮어씁니다.
"a"추가 (append)파일 끝에 내용을 이어씁니다. 파일이 없으면 새로 만듭니다.
⚠ 주의 — "w" 모드

"w" 모드로 열면 기존 파일의 내용이 전부 지워집니다. 기존 내용을 보존하면서 새 내용을 추가하려면 반드시 "a" 모드를 사용하세요.

3.with 문 — 파일을 안전하게 닫는 법

파일을 열면 반드시 닫아야 합니다. 파일을 열어두면 메모리를 낭비하고, 다른 프로그램이 그 파일을 사용하지 못할 수 있습니다. open() 후 close()를 수동으로 호출해도 되지만, 실수로 빠뜨리면 문제가 생깁니다.

파이썬은 이 문제를 with 문으로 우아하게 해결합니다. with 블록이 끝나면 파일이 자동으로 닫힙니다. 오류가 발생해도 마찬가지입니다.

python
1234567
# ❌ 수동 방식 — close()를 깜빡할 수 있음
f = open("note.txt", "r", encoding="utf-8")
content = f.read()
f.close()  # 잊으면 안 됨!

# ✅ with 방식 — 블록 끝에서 자동으로 닫힘
with open("note.txt", "r", encoding="utf-8") as f:
    content = f.read()
with open("note.txt", "r") as f :← 파일을 열고 f 라는 이름으로 사용
content = f.read()← 파일 내용을 읽어 content에 저장
↑ 들여쓰기가 끝나는 순간 파일이 자동으로 닫힙니다.
💡 TIP — with 문은 습관처럼

파일을 다룰 때는 항상 with 문을 사용하세요. 실무 파이썬 코드의 99%는 with open(...) as f: 형태를 사용합니다. 처음부터 이 패턴에 익숙해지는 게 좋습니다.

4.파일에 쓰기 — "w"와 "a"

"w" 모드 — 새로 쓰기

파일에 내용을 새로 쓸 때는 "w" 모드를 사용합니다. write() 메서드에 문자열을 전달하면 됩니다.

python
123456
# greeting.txt 파일에 내용 쓰기
with open("greeting.txt", "w", encoding="utf-8") as f:
    f.write("안녕하세요!\n")
    f.write("파이썬 파일 입출력을 배우고 있습니다.\n")

print("파일 작성 완료!")
실행 결과 (콘솔)
파일 작성 완료!

이 코드를 실행하면 같은 폴더에 greeting.txt 파일이 생성됩니다. 파일 안에는 아래와 같이 저장됩니다.

greeting.txt 파일 내용
안녕하세요!
파이썬 파일 입출력을 배우고 있습니다.
⚠ write()는 줄바꿈을 자동으로 넣지 않음

print()와 달리 write()는 줄 끝에 \n을 자동으로 추가하지 않습니다. 줄을 바꾸려면 직접 \n을 써주세요.

여러 줄 한 번에 쓰기 — writelines()

리스트의 각 줄을 파일에 한 번에 쓰고 싶을 때는 writelines()를 사용합니다. 단, 줄바꿈 문자는 역시 직접 포함시켜야 합니다.

python
1234567
lines = [
    "첫 번째 줄\n",
    "두 번째 줄\n",
    "세 번째 줄\n",
]
with open("lines.txt", "w", encoding="utf-8") as f:
    f.writelines(lines)  # 리스트의 모든 문자열을 한 번에 씀

"a" 모드 — 이어쓰기 (append)

기존 파일의 내용을 지우지 않고 끝에 추가하려면 "a" 모드를 사용합니다. 일기장에 날마다 새 내용을 덧붙이는 것과 같습니다.

python
12345678910
# 첫 실행: 파일 생성 후 내용 작성
with open("log.txt", "w", encoding="utf-8") as f:
    f.write("[1일차] 파이썬 시작!\n")

# 두 번째 실행: 기존 내용 유지하며 끝에 추가
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("[2일차] 파일 입출력 배움!\n")

# 세 번째 실행: 또 추가
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("[3일차] 예외처리도 배울 예정!\n")
log.txt 최종 내용
[1일차] 파이썬 시작!
[2일차] 파일 입출력 배움!
[3일차] 예외처리도 배울 예정!

5.파일 읽기 — read(), readline(), readlines()

파일을 읽는 방법은 세 가지가 있습니다. 파일의 크기와 사용 목적에 따라 골라 씁니다.

① read() — 전체를 문자열로

파일 전체 내용을 하나의 문자열로 읽어옵니다. 파일이 작을 때 간편합니다.

python
1234
with open("greeting.txt", "r", encoding="utf-8") as f:
    content = f.read()  # 파일 전체를 하나의 문자열로

print(content)
실행 결과
안녕하세요!
파이썬 파일 입출력을 배우고 있습니다.

② readlines() — 줄별 리스트로

파일의 각 줄을 원소로 하는 리스트를 반환합니다. 각 줄 끝에 \n이 포함되어 있으니 주의하세요.

python
1234567
with open("greeting.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()  # ['안녕하세요!\n', '파이썬 ...']

print(lines)
print(f"총 {len(lines)}줄")

# strip()으로 줄 끝 \n 제거
for line in lines:
    print(line.strip())
실행 결과
['안녕하세요!\n', '파이썬 파일 입출력을 배우고 있습니다.\n']
총 2줄
안녕하세요!
파이썬 파일 입출력을 배우고 있습니다.

③ for 루프로 직접 순회 — 가장 실전적인 방법

파일 객체는 그 자체로 반복 가능합니다. readlines()를 쓰지 않고 for 루프로 바로 줄을 읽으면 메모리를 훨씬 절약할 수 있습니다. 파일이 클수록 이 방법이 유리합니다.

python
1234
with open("log.txt", "r", encoding="utf-8") as f:
    for line in f:             # 파일을 한 줄씩 직접 순회
        line = line.strip()  # 양쪽 공백 및 \n 제거
        print(line)
실행 결과
[1일차] 파이썬 시작!
[2일차] 파일 입출력 배움!
[3일차] 예외처리도 배울 예정!
💡 strip() vs rstrip()

strip()은 양쪽 공백과 줄바꿈을 모두 제거합니다. 줄 끝 \n만 제거하고 싶다면 rstrip()을 사용하세요. 파일을 줄 단위로 읽을 때 습관처럼 .strip()을 붙여두면 편합니다.

6.파일이 없을 때 — FileNotFoundError

존재하지 않는 파일을 "r" 모드로 열려고 하면 FileNotFoundError가 발생합니다.

python
12
with open("없는파일.txt", "r", encoding="utf-8") as f:
    content = f.read()
오류 메시지
FileNotFoundError: [Errno 2] No such file or directory: '없는파일.txt'

이 오류는 다음 챕터에서 배우는 try / except로 처리할 수 있습니다. 지금은 오류가 발생하는 상황과 오류 이름만 기억해두세요.

python
123456
# try/except로 파일이 없는 경우를 처리 (맛보기)
try:
    with open("없는파일.txt", "r", encoding="utf-8") as f:
        content = f.read()
except FileNotFoundError:
    print("파일을 찾을 수 없습니다.")
실행 결과
파일을 찾을 수 없습니다.

7.인코딩 — 한글 파일을 다룰 때 필수

컴퓨터는 모든 것을 숫자로 저장합니다. 문자를 숫자로 대응시키는 약속이 인코딩(encoding)입니다. 한글을 포함하는 파일은 반드시 올바른 인코딩을 지정해야 글자가 깨지지 않습니다.

인코딩설명언제 쓰나
"utf-8"세계 표준. 한글·영어·이모지 모두 표현새 파일 만들 때 항상 사용 권장
"cp949"윈도우 한글 인코딩 (EUC-KR 계열)윈도우에서 만든 오래된 한글 파일 열 때
💡 TIP — 항상 encoding="utf-8"

새로운 파일을 만들 때는 항상 encoding="utf-8"을 명시하세요. 지정하지 않으면 운영체제의 기본값이 사용되는데, 윈도우에서는 cp949가 기본값이라 한글이 깨질 수 있습니다.

03. "a" 모드로 파일 끝에 한 줄 더 추가한 뒤, 파일 전체를 다시 읽어 총 줄 수를 출력합니다.
도전. 사용자에게 input()으로 메모를 입력받아 파일에 저장하는 프로그램을 만들어보세요. 프로그램을 실행할 때마다 새 메모가 파일 끝에 추가되어야 합니다.
▸ 핵심 요약
open(파일, 모드) 파일을 열어 파일 객체를 반환. 반드시 encoding="utf-8"을 명시. with … as f 파일을 안전하게 열고, 블록이 끝나면 자동으로 닫음. 실무 표준 패턴. "r" / "w" / "a" 읽기 / 새로쓰기(덮어씀) / 이어쓰기. "w"는 기존 내용을 모두 지움에 주의. f.write(문자열) 파일에 문자열 씀. 줄바꿈은 자동이 아니므로 \n을 직접 포함해야 함. f.read() 파일 전체를 하나의 문자열로 반환. for line in f 파일을 한 줄씩 순회. 대용량 파일에서 메모리 효율적. .strip()으로 \n 제거. FileNotFoundError 존재하지 않는 파일을 읽으려 할 때 발생. try/except로 처리(다음 챕터).
