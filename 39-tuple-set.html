튜플과 셋 — 잠긴 리스트, 중복 없는 바구니

리스트·딕셔너리에 이어 파이썬의 나머지 두 핵심 자료구조를 배웁니다. 튜플은 한번 만들면 바꿀 수 없는 리스트이고, 셋은 중복을 자동으로 제거하는 집합입니다. 언제, 왜 쓰는지를 이해하면 코드가 훨씬 명확해집니다.

▸ 이 챕터를 마치면
  • 튜플과 리스트의 차이를 설명하고 상황에 맞게 선택할 수 있다
  • 튜플 언패킹으로 여러 변수에 한 번에 값을 담을 수 있다
  • 셋으로 중복을 제거하고 집합 연산(합집합·교집합·차집합)을 수행할 수 있다
  • 리스트·튜플·셋·딕셔너리를 상황에 맞게 선택할 수 있다

PART 1.튜플 (Tuple)

1.튜플이란? — 잠긴 리스트

튜플은 리스트와 거의 똑같습니다. 단 하나의 차이가 있는데, 한 번 만든 뒤에는 내용을 바꿀 수 없습니다. 자물쇠가 채워진 리스트라고 생각하세요.

PYTHON — 리스트 vs 튜플
# 리스트: [] 사용, 수정 가능
fruits_list = ["사과", "바나나", "포도"]
fruits_list[0] = "딸기"   # OK

# 튜플: () 사용, 수정 불가능
fruits_tuple = ("사과", "바나나", "포도")
# fruits_tuple[0] = "딸기"  # TypeError: 수정 불가!

print(fruits_tuple[1])   # 바나나  (읽기는 됩니다)
print(len(fruits_tuple)) # 3
📌 튜플을 쓰는 이유

① 실수로 바꾸면 안 되는 데이터를 보호 — 좌표, 날짜, 설정값처럼 고정되어야 하는 값
② 딕셔너리의 키로 사용 가능 — 리스트는 키가 될 수 없지만 튜플은 됩니다
③ 함수에서 여러 값 반환 — 파이썬 함수는 튜플로 여러 값을 한 번에 반환합니다

2.튜플 만들기

PYTHON
 1# 기본 생성
 2point   = (3, 7)           # x=3, y=7 좌표
 3rgb     = (255, 128, 0)    # R, G, B 색상값
 4today   = (2026, 3, 5)    # 연, 월, 일
 5
 6# 괄호 생략도 됩니다 (콤마가 핵심!)
 7colors = "빨강", "파랑", "초록"
 8print(type(colors))  # <class 'tuple'>
 9
10# 원소가 1개인 튜플 — 콤마 필수!
11single_wrong  = ("하나")    # str! 튜플 아님
12single_right  = ("하나",)   # tuple  ← 콤마 추가
13print(type(single_wrong))  # <class 'str'>
14print(type(single_right))  # <class 'tuple'>
15
16# 빈 튜플
17empty = ()
⚠️ 원소 1개 튜플의 함정

("하나")는 그냥 괄호로 감싼 문자열입니다. 원소가 하나인 튜플을 만들려면 반드시 뒤에 콤마를 붙여야 합니다: ("하나",)

3.튜플 언패킹 — 한 번에 여러 변수에 담기

튜플의 가장 강력하고 자주 쓰이는 기능입니다. 튜플 안의 값을 여러 변수에 한 줄에 나눠 담을 수 있습니다.

PYTHON
 1# 기본 언패킹
 2point = (3, 7)
 3x, y = point
 4print(f"x={x}, y={y}")   # x=3, y=7
 5
 6# 함수 반환값 언패킹 — 실전에서 매우 자주 씁니다
 7def min_max(numbers):
 8    return min(numbers), max(numbers)  # 튜플로 반환
 9
10data = [4, 1, 9, 2, 7]
11lo, hi = min_max(data)
12print(f"최솟값: {lo}, 최댓값: {hi}")  # 최솟값: 1, 최댓값: 9
13
14# 변수 값 맞바꾸기 — 파이썬만의 깔끔한 스왑!
15a, b = 10, 20
16a, b = b, a          # temp 변수 없이 교환
17print(a, b)           # 20 10
18
19# for문에서 items()와 함께 (이미 써봤죠?)
20scores = {"수학":90, "영어":85}
21for subject, score in scores.items():  # items()가 튜플을 반환합니다!
22    print(f"{subject}: {score}")
▸ 실행 결과
x=3, y=7 최솟값: 1, 최댓값: 9 20 10 수학: 90 영어: 85
💡 딕셔너리 .items()가 사실은 튜플이었습니다

지난 챕터에서 for k, v in d.items()를 쓸 때 이미 튜플 언패킹을 사용하고 있었습니다. items()는 ("키", "값") 형태의 튜플 묶음을 반환하기 때문입니다.

4.튜플 주요 기능

튜플은 수정이 안 될 뿐, 읽기 관련 기능은 리스트와 동일합니다.

PYTHON
 1t = (10, 20, 30, 20, 10)
 2
 3print(t[0])          # 10  — 인덱스 접근
 4print(t[1:4])        # (20, 30, 20)  — 슬라이싱
 5print(len(t))        # 5  — 길이
 6print(t.count(20))   # 2  — 20의 개수
 7print(t.index(30))   # 2  — 30의 위치
 8print(30 in t)       # True — 포함 여부
 9
10# 리스트 ↔ 튜플 변환
11lst = list(t)         # 튜플 → 리스트 (수정 가능하게)
12tpl = tuple(lst)     # 리스트 → 튜플 (고정)

PART 2.셋 (Set)

5.셋이란? — 중복 없는 바구니

셋은 중복된 값을 허용하지 않는 자료구조입니다. 수학의 집합과 똑같습니다. 순서가 없고, 같은 값이 두 번 들어가도 하나만 남습니다.

PYTHON — 셋의 핵심 특징
# 중복이 자동으로 제거됩니다
numbers = {1, 2, 3, 2, 1, 4}
print(numbers)   # {1, 2, 3, 4}  — 순서는 실행마다 다를 수 있음

# 가장 흔한 활용: 리스트 중복 제거
tags = ["파이썬", "코딩", "파이썬", "AI", "코딩"]
unique_tags = list(set(tags))
print(unique_tags)   # ['파이썬', '코딩', 'AI']  (순서는 보장 안 됨)
📌 셋의 세 가지 특징

① 중복 없음 — 같은 값은 하나만 저장
② 순서 없음 — 인덱스로 접근 불가 (s[0] 오류)
③ 빠른 검색 — in 연산이 리스트보다 훨씬 빠름 (데이터가 많을수록 효과적)

6.셋 만들기와 기본 조작

PYTHON
 1# 생성 — 중괄호 사용 (딕셔너리와 다르게 콜론이 없음)
 2fruits = {"사과", "바나나", "포도"}
 3
 4# 빈 셋 — 반드시 set()을 써야 합니다
 5empty_dict = {}          # 이건 빈 딕셔너리!
 6empty_set  = set()       # 올바른 빈 셋
 7
 8# 추가 — add()
 9fruits.add("딸기")
10fruits.add("사과")       # 이미 있으면 무시됨
11print(fruits)            # {'사과', '바나나', '포도', '딸기'}
12
13# 삭제 — remove() vs discard()
14fruits.remove("바나나")   # 없으면 KeyError!
15fruits.discard("망고")   # 없어도 오류 없음 (안전)
16
17# 포함 여부 — in (매우 빠름)
18print("사과" in fruits)   # True
19print(len(fruits))       # 현재 원소 수
⚠️ 빈 셋 함정

{}는 빈 딕셔너리입니다. 빈 셋은 반드시 set()으로 만들어야 합니다.

7.집합 연산 — 합집합 · 교집합 · 차집합

셋의 가장 강력한 기능입니다. 두 집합 사이의 관계를 단 한 줄로 구할 수 있습니다.

|
합집합 (UNION)
A | B
A.union(B)
두 집합의 모든 원소
&
교집합 (INTERSECTION)
A & B
A.intersection(B)
공통 원소만
-
차집합 (DIFFERENCE)
A - B
A.difference(B)
A에만 있는 원소
^
대칭차집합 (XOR)
A ^ B
A.symmetric_difference(B)
한쪽에만 있는 원소
PYTHON
 1python_students = {"Alice", "Bob", "Carol", "Dave"}
 2java_students   = {"Bob", "Dave", "Eve", "Frank"}
 3
 4# 합집합 — 두 수업 중 하나라도 듣는 학생
 5print(python_students | java_students)
 6# {'Alice', 'Bob', 'Carol', 'Dave', 'Eve', 'Frank'}
 7
 8# 교집합 — 두 수업 모두 듣는 학생
 9print(python_students & java_students)
10# {'Bob', 'Dave'}
11
12# 차집합 — 파이썬만 듣고 자바는 안 듣는 학생
13print(python_students - java_students)
14# {'Alice', 'Carol'}
15
16# 대칭차집합 — 한 수업만 듣는 학생
17print(python_students ^ java_students)
18# {'Alice', 'Carol', 'Eve', 'Frank'}
💡 집합 연산의 실전 활용

두 리스트의 공통 항목을 찾거나, 한쪽에만 있는 항목을 추려낼 때 셋이 리스트 이중 루프보다 훨씬 빠르고 코드도 짧습니다.

8.실전 예제 — 출석 체크 & 중복 제거

PYTHON
 1# 예제 1: 출석 체크 — 결석자 찾기
 2all_students  = {"김파이", "이코딩", "박루프", "최클래스", "정함수"}
 3attended      = {"김파이", "박루프", "정함수"}
 4
 5absent = all_students - attended
 6print(f"결석자: {absent}")
 7# 결석자: {'이코딩', '최클래스'}
 8
 9# 예제 2: 방문자 로그 중복 제거 후 순위 집계
10visit_log = [
11    "user_A", "user_B", "user_A", "user_C",
12    "user_B", "user_A", "user_D", "user_C"
13]
14unique_visitors = set(visit_log)
15print(f"전체 방문 횟수: {len(visit_log)}")
16print(f"순 방문자 수:   {len(unique_visitors)}")
17
18# 예제 3: 두 리스트의 공통 항목
19wish_list  = ["노트북", "키보드", "마우스", "모니터"]

23print(f"지금 살 수 있는 항목: {can_buy}")
24# 지금 살 수 있는 항목: {'마우스', '모니터'}

결석자: {'이코딩', '최클래스'} 전체 방문 횟수: 8 순 방문자 수: 4 지금 살 수 있는 항목: {'마우스', '모니터'}

9.네 가지 자료구조 총정리 — 언제 뭘 쓸까?

📋 리스트 [ ]
✅순서 있음, 중복 허용
✅수정·추가·삭제 자유
📌순서 있는 데이터 목록
📦 튜플 ( )
✅순서 있음, 중복 허용
🔒수정 불가 (읽기 전용)
📌고정값, 좌표, 함수 반환
🔵 셋 { }
🚫순서 없음, 중복 불가
✅추가·삭제 가능
📌중복 제거, 집합 연산
🗂️ 딕셔너리 { key: value }
✅키:값 쌍으로 의미 있는 데이터
✅키로 빠른 조회
📌속성이 여러 개인 객체, JSON 데이터
자료구조기호순서중복수정대표 사용 예
리스트[ ]✅✅✅성적 목록, 장바구니
튜플( )✅✅❌좌표, RGB, 날짜
셋{ }❌❌✅태그, 방문자, 공통항목
딕셔너리{k:v}✅키 불가✅학생 정보, 설정값

10.자주 발생하는 오류와 해결법

❌
튜플 수정 시도 — TypeError: 'tuple' object does not support item assignment
튜플은 읽기 전용입니다. 수정이 필요하면 list()로 변환 후 수정하고 다시 tuple()로 변환하세요.
❌
셋 인덱스 접근 — TypeError: 'set' object is not subscriptable
s[0] 불가. 순서가 없으므로 인덱스 접근이 없습니다. 순서가 필요하면 sorted(s)로 정렬 후 리스트로 변환하세요.
❌
빈 셋을 {}로 생성 — 딕셔너리가 됩니다.
빈 셋은 반드시 set()을 사용하세요.
❌
원소 1개 튜플 실수 — (42)는 정수 42와 동일합니다.
(42,)처럼 콤마를 꼭 붙이세요.
❌
셋에 리스트 추가 시도 — TypeError: unhashable type: 'list'
셋의 원소는 변경 불가능한 타입(int, str, tuple)만 가능합니다. 리스트는 원소가 될 수 없습니다.
📌 핵심 요약
튜플 () — 수정 불가 리스트
원소 1개 튜플은 (x,) 콤마 필수
언패킹: x, y = (3, 7)
변수 스왑: a, b = b, a
셋 {} — 중복 없음, 순서 없음
빈 셋은 set() (중요!)
add() / discard() — 추가/삭제
| 합집합, & 교집합, - 차집합
중복 제거: list(set(lst))
셋은 인덱스 접근 불가
